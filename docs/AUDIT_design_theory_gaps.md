## 1. 三路共振的第一洞：「案没跑成」在三层都没有名字

同一个洞贯穿理论、设计、实现，是本次审计最重要的发现：

| 层 | 洞的形态 | 证据 |
|---|---|---|
| 理论 | exec 被写成全函数（`O=Exec(P)+ε`，ε 只建模观测噪声）；42 条公式+S 六定理无任何条目覆盖「σ 第 k 步失败后，后续步在什么状态上执行、断言真值是否有定义」 | THEORY_k_state_machine.md:18,913；τ∘exec≈id 同假设 |
| 设计 | verdict 事实 schema 只有二值 `pass\|fail`（§2.1 L44）——ERROR≠FAILURE 在 schema 层就写死缺失 | DESIGN §2.1 |
| 实现 | **采集侧五值早已存在**（`run_case.py:73` pass/fail/error/busy/unknown；digest 的 stale_log→unknown 并注明"绝不采信"）——被入账层一行折叠成二值：`nodes.py:586` | batch_tools.py:655-699 |
| 实弹 | 668030 恢复步失败→not_found 空真通过（vacuous pass）；文件级崩溃后级联 unknown 全部以 fail 入账→假签名→误 frozen→假归因烧轮 | run13 取证 |

**修法（一件事三层同修）**：理论补「exec 部分函数化+断言有效性公理」（R 仅在支配动作步全部 ok 时有真值，否则案=broken 第三态）；设计 verdict schema 增 `broken/not_run`；实现 reconcile 保真透传采集侧已有的五值，fold 派生 S_BROKEN——broken 案不计 fail 签名（防误 frozen）、归因短路为级联受害、报告单列分母。

## 4. xUnit 语义对标（缺失 7 / 部分 11 / 等价 5 / 超集 1）

**结构性结论**：远端框架是「前 xUnit」形态（1 pytest item=整本 xlsx，案级语义手搓 for 循环）——运行时语义只能在采集归因面补。本项目已把**静态可判部分**做到等价甚至反超（报告保真、collection 预检、run-identity），缺失全部集中在**四个运行时相位**：ERROR 态、teardown 成功性、case 级 timeout、setup 相位。

缺失 Top5（按实弹风险）：
1. **verdict 三值直通入账**（杠杆最大改动最小——采集侧五值已在，nodes.py:586 一行折没）
2. **恢复步成功性验证**（teardown verdict：digest 机械扫恢复步回显错误 markers→broken+床账标记——封 668030 型实弹的机制位）
3. **case 级 timeout 判定**（心跳已能定位当前案，加同案超阈→timeout_suspect 事实——T1 三症状机械可分）
4. **setup 相位判定**（init 段回显扫 CLI error→setup_failed 事实→归因禁 V 层同向重编）
5. **known_defect→xfail/xpass 闭环**（已知缺陷摊销归因成本；xpass=缺陷修复/判例陈腐的唯一自动信号）

Top4 本质是同一件事的四个切面：**给「案没跑成」一个与「案跑红了」不同的名字，并让它一路活到台账。**

## 6. 方法论结论（回答「理论到底犯了什么错」）

1. **理论不生成判定器，坑生成判定器**——转化率 20% 完整机械化是死循环的量化形态。公式写完的验收标准应当是「谁消费它」，无消费者的公式按 C14 纪律降为注记。
2. **全函数假设是系统性思维缺陷**（理论 1 处+设计实现 7 处）——每一层都假设自己脚下那层不会失败。修复不是逐个打补丁，是立一条设计红线：**任何跨进程/跨设备/跨文件的动作必须定义失败语义，「静默继续」不是合法选项。**
3. **「机械强制」的字样必须配三件套**：判定代码+失败语义+测试锚。缺一即回写设计降级措辞。本次 11 条声称-落差全部源于字样先行于实现。
4. **成熟范式对标是抄作业不是选修课**——infra 理论（对标产物）转化率 47% 最低，恰说明抄完没做比不抄更危险：它制造了「已覆盖」的幻觉。
