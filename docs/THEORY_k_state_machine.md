# 理论：用例正确性的三合取模型与 K 状态机

> 状态：理论基座 v1（2026-07-09）。本文沉淀 2026-07-08/09 两日对话推导 + 对抗性反驳后的存活结论。
> 证据基础：全部锚定一手设备实证（"金牌数据"，见 §10 证据索引）——pe1 / selfheal1 / selfheal2 三批
> 同脑图对照、五案手动上机裁决、exp_recheck5 / digprobe / D3 / E5a 定向实验、942 对 grade 配对、511 卷等价反扫。
> 用法：任何新机制立项前先过本文 §2 三合取与 §4 状态机——落得进=不需要新概念；落不进=对理论的反例，先修理论。

## 1. 核心对象与记号

| 记号 | 含义 | 访问方式 |
|---|---|---|
| **R\*** | 设备真实语义（ground truth） | 只能经设备访问，且**分层可查询**（§3.2） |
| **R_spec** | 设计规格（手册/配置文档声称的语义） | 文本，可离线读 |
| **K** | 知识库 = **K_is ⊔ K_ought**（双层，§2.6）：K_is = 先例卷可跑形态 ∪ CLI 手册 ∪ footprint ∪ known_defects；K_ought = spec/design ∪ 需求单 ∪ bug 裁决史 ∪ 人写历史 excel 的验证意图 ∪ 欠定决策史 ∪ app 手册 | 检索；K_is 是对 R\* 的部分近似，K_ought 载**异步的人类意图** |
| **C = (D, P, E)** | 测试用例 = 描述（意图）+ 配置过程 + 预期结果 | 编译产物 |
| **O** | 上机观测 = Exec(P) + ε | ε 为观测通道噪声（实证存在，§3.1 第④类） |
| **锚 a** | (build 指纹, 实证时间 run_ts, 血统 lineage) | lineage ∈ {manual, device, precedent} |

D 的本质：**人对意图的有损自然语言投影**，混叠"意图本体"与"实现提示"（608 的 disable 是提示、down 是意图）。但对抗反驳（§9-C2）表明：不应对 D 做集中规范化——分布式解码的多样性冗余（570 解对/608 解错→即时写回流动）优于单点解码的审计性。

## 2. 正确性三合取（中心公式）

一条用例**正确**是三个合取同时成立：

```
Sound(C) ≝ (P ⊩ D) ∧ (E ≅ proj(D)) ∧ (R*(P) ⊨ E)
            ─────①      ─────②           ─────③
  ① 配置实现描述的场景（不多配、不少配、不变形）
  ② 预期是描述在观测空间的忠实投影（不弱化、不翻转、不越权）
  ③ 设备执行配置的行为满足预期
```

**上机验证只测合取③**：`Verify(C) = [O ⊨ E]`。这是系统最重要的结构事实——

> **验证盲区定理（实证）**：O ⊨ E 无论重复多少次，推不出 ① 与 ②。
> 一切"上机 PASS 的假用例"都住在这个盲区里。

金牌实证：644 删 AAAA 断言（②被弱化，PASS）；首批 608/570 毒卷 E 反写（②被翻转，PASS 且入库成"金标准"）；493 多挂 CNAME 池（①变形，直接改变被测行为）；035373 not_found→found 迁就（②翻转）。

**盲区的分布（两轮修正后）**：21 张交付卷机械保真审计（§6.5.1）查出 2 案漏维——644（压力路径：
frozen override 删断言）与 **373（R1 首轮即漏**：复杂流程 case 的 AAAA 半维静默丢失，两批复发）。
即失真有两种机理：压力驱逐（重编时删掉会 fail 的覆盖）与注意力占据（首轮流程复杂时漏维）。
两者同被 intent↔卷面投影比对（词面显式子集）覆盖；压力路径另需**单调性检查**（删除必须显式）：

```
Monotonic(C_new | C_old) ≝ ObsCmds(C_new) ⊇ ObsCmds(C_old) ∧ Asserts(C_new) ⊇ σ(Asserts(C_old))
   （重编产物的观测命令集与断言集相对旧卷只准增/等价变换，删除型 diff 必须显式声明并过审）
```

这是集合差集判定，纯机械。合取②的**全量** oracle 不可判定（§9-C1，撞 grade 之死），单调性是它在压力路径上的可判定投影。

**合取③的语境参数（2026-07-10 yzg 终验实证）**：③ 的完整形式是 R\*(P, ctx) ⊨ E——执行语境
ctx（卷面组成、执行顺序、**床态向量 B**——本机磁盘持久层/对端节点态/分区文件系统/床级
互斥态，2026-07-10 手册取证四通道）是量化变元，不是背景常量。ctx = (π, B)：π 是卷内排列
（只影响通道①），B 是跨案/跨设备/跨批的持久床态；锚差监控 Δ（公式7）的适用面由 build
一维扩到床态多维。"pass" 因此是**二元谓词
Pass(c, ctx)**；交付判据是 Pass(c, ctx_delivery)（整卷连跑）。把它投影成一元 Pass(c) 等于把
量词从「交付语境下成立」偷换成「∃ 某语境成立」——投影无损的充要条件是 **case 间独立性假设**
（per-case 清场 ⇒ 任意语境同结果）。该假设的边界在其自己的声明里：清场只清*运行配置*；
write file/net/all 的产物是*盘上文件*，跨案存活。yzg 保存族三案（668015/030/044）实证：
子集语境 pass ∧ 整卷语境 fail——∃-pass ≠ 交付-pass，投影在假设边界外失真。

**互扰消解推论（2026-07-10 渲染层设计两轮对抗定形——行动的理论根）**：设 δ(c) 为 case c
在 Λ（per-case 清场）之外存活的持久写集。case 间独立性假设的精确形式是 ∀c: δ(c)=∅。互扰的
**消解手段是复合施加**：案内自清理（c 在案尾清空 δ(c)——唯一使独立性假设重新成立的解，对全部
后继案同时生效）**∘** 排卷尾（兜底自清理覆盖不到的残余）。「接受单跑」**不在消解集合中**——它是
把 Pass(c, ctx_delivery) 偷换为 ∃-pass 的量词投影（本节上文的失真形式），只能作为用户显式止损
裁决存在，且必须标注不符交付预期、记入未通过卷。
**目标函数与终态（2026-07-10 三轮对抗定形）**：引擎目标 = 脑图每一案编译成真通过的自动化
用例。可导性定理保证 fail 有限步归到唯一根因；Ω 五类中**只有③产品缺陷是案侧无修法的类**——
故唯一非通过的正当终态 = 缺陷候选（四查+换形态坐实）；其余 fail 全是中间态，引擎必须继续
（挂起是资源界的暂停不是终态，事实流断点续跑跨批有效）。「接受单跑」不是引擎出口——只在用户
主动输入时被接受。
**Ask 充要条件 → §2.6 Ask 定理（四稿定形，此处只留演化史）**：一稿三选项菜单（把定理退化为
投票）→ 二稿菜单生成器（仍是选择题）→ 三稿"意图权/资源权"二分+修法队列空门 → 四稿用户驳正
定形：**"手册 vs 实机不符"不是实验可裁的**——实验只能裁"差异存在"（实然），裁不出"谁代表真意"
（应然），三稿把它错标为"不问"。终版分界是实然/应然（is/ought），资源权归并入应然（所有权/授权
皆规范命题），完整定理、K_ought 检索次序、收敛律与对抗检查见 §2.6。三稿的有效构件保留：修法
队列空门（=实然手段试尽的机械面）、缺陷四查门、「接受单跑」仅收用户主动输入。yzg 实证支持：030 的终验翻挂恰发生在尾序前驱 015 换
卷之轮（CLI Ch20：clear sdns all 不清盘上文件，需显式文件级清理）——015 复合修法后 030 应随之
稳定，一个修法救两案（可证伪预测，下批检验）。
**推论（对验证盲区定理的补充）**：oracle 裁决的效力 = min(oracle 强度, 消费链最弱环节)。
上机是唯一语义权威，但其裁决若未被状态机消费（yzg：终验 3 fail 落盘后从未被读），
效力为零——盲区不仅在「测什么」（①②测不到），也在「测了之后谁听」。

## 2.5 信息量三分解：I(C) = I_G ⊔ I_E ⊔ I_V

### 2.5.1 定义

一条断言 a 的**信息量**取信息论定义：

```
I(a) = -log₂ P(O ⊨ a | 行为无关假设)
   恒真断言: P = 1 → I = 0        （zero_info_assertion 原理的信息论本体）
   恒假断言: P = 0 → I 无定义      （必崩门拦截对象）
```

> **测度的诚实声明（对抗检查修正）**："行为无关假设"的参照分布未定义，故 I 的**数值**不可算——
> 本理论只使用它的**序结构**：I=0 可机械判定（恒真族），I>0 之间不比大小。实际运转的是二值
> {零信息, 正信息} + 维度计数 dim_V。公式的价值是把 zero_info 从启发式变成定义的推论，不是提供数值。

断言按观测算子性质归层（layer(a) ∈ {G, E, V}，机械判定已实现：observe_kind + F 列算子）：

```
I(C) = ( I_G, I_E, I_V ) ∈ ℝ³₊       —— 信息量不是标量，是分层向量
```

**⊔（不交并）的确切含义——两条不可通约公理**：

1. **不可补偿**：G/E 信息量堆得再高也不产生 V 信息量（配置检查写一百条也不验证任何行为）。
   实证：588691 恒真族卷——G 段"信息丰富"、I_V = 0、当年照样 PASS 入库。
2. **用途不同**：V = **覆盖信息**（意图的行为验证，决定这条用例存在的价值）；
   G/E = **诊断信息**（fail 时定位根因层——分层判定树的"配置生效了吗"一步消费的正是 G 段）。
   G/E 信息量的价值在 fail 路径上兑现，V 的价值在 PASS 路径上兑现——两类信息在不同事件下变现，故不可交换。

### 2.5.2 覆盖判据与守恒律

意图 D 的信息量需求主要在 V 维（D 的每个行为断言张一个 V 维度）：

```
覆盖判据:  dim_V(C) ≥ dim_V(D)          （Cov = dim(span(A)∩B)/dim(B) 的直和表述）
守恒律:    重编/override 不得使 ΔI_V < 0  （单调性检查的信息论内容）
```

644 假 PASS 的定量刻画：意图 dim_V(D)=3（CNAME/A/AAAA 三类行为断言），删 AAAA 后 dim_V(C)=2 < 3——
但 Verify 只测 [O ⊨ E]（总体可满足性），对 I_V 维度亏损盲。**验证盲区定理的定量版**：
PASS 信号携带的信息 = "现有断言全部成立"，不携带 "断言集张满意图空间"。

### 2.5.3 可计算代理（理论量 → 已有估计器的映射）

不发明不可算的测度——每个理论量给出已实现/可实现的估计器：

| 理论量 | 可计算代理 | 实现状态 |
|---|---|---|
| I(a) = 0（零信息） | count_tautology / is_config_existence_check / 恒真锚点族 | ✅ 必崩门+原理层 |
| layer(a) | observe_kind + F 列算子 + matched_config | ✅ observe_ops |
| dim_V(C) | genuine_v_count（真 V 断言计数） | ✅ grade_extract |
| dim_V(D) | **无统一代理**（对抗检查 A2：expected 粒度不均——644 的"返回service ip"是一条 expected 却含 A/AAAA 两维，按条计数抓不住标志性违例）；可判定子集=结构化投影（记录类型/观测对象词面） | ⚠ 粒度问题是 C1 的另一面 |
| ΔI_V ≥ 0（守恒） | 重编前后 ObsCmds/Asserts 集合差（单调性检查） | ✅ 单调门（compile_emit，2026-07-09）：旧卷为基线，观测动词类+DNS 记录类型集合不减；`coverage_reduction_reason` 显式声明放行+记 `monotonicity_violation` 信号 |
| dim_V 的意图侧缺口 | intent_record_type_gap（词面显式子集） | ✅ 本轮（覆盖面见 C1） |
| K 条目信息量为负（误导） | stale/poisoned 标注（§5） | 部分 |

> K 条目同样有信息量与符号：verified 条目 I > 0（约束 R\* 的猜测空间）；stale/poisoned 条目 **I < 0**
> （主动误导——044572 被一条可疑缺陷记录带偏即负信息量变现）。污染检测的信息论意义：把库的净信息量
> Σ I(k) 保持为正——这就是"K 运维"的目标函数。

## 2.6 Ask 定理：实然/应然分界、K_ought 检索与收敛律（2026-07-10 四稿定形）

### 2.6.1 分界与权威

**实然（is）**：关于系统当前状态的可观测命题——设备对 X 回了什么、文档写了什么、卷面长什么样、
库里记载了什么。全部可由 {读文档, 查 K, 设备实验, 计算} 有限步决定（可导性定理管辖）。
**应然（ought）**：关于"该是什么"的规范命题——该测什么、该期望什么、该用什么方法验证、这个行为
算不算对、该不该动别人的资源。**实验永远建立不了应然**：它能证明"设备做了 Y"，不能证明"Y 是产品
该有的行为"——is-ought 鸿沟，正是 observe-then-assert 红线的理论本体。

**权威结构**：每个制品都是某人意图的投影——D=用例作者的测试意图；手册/spec/design=研发的产品
意图；R\* =实现（可偏离设计=缺陷）。应然的权威=意图所有者（用例作者/研发，经用户代理通道触达）。
两个投影冲突时，冲突在意图之间，只有所有者能仲裁；引擎硬选任何一边都是改写某人的意图（照实机写
断言=改写测试意图且缺陷从此测不出；照手册写=用例永挂在可能无错的设备上）。

**同形判据（开放列表，不枚举场景）**：手册↔实机、脑图预期↔上机结果、验证方法↔功能实现形态、
编排假设↔持久化语义……凡「A 源说 X、B 源说 Y、选边即改写某方意图」即中此判据。新冲突形态=
新数据实例，不需要新理论。

### 2.6.2 K_ought 检索先于 ask（异步的人先于同步的人）

自然人卡住时先翻 spec、翻旧用例、搜 bug 史——那是在检索**异步的人**；都翻不出才去敲门。故
ask 不是应然的第一通道，是**检索余项**：

```
应然决策 → 检索 K_ought（次序:spec/design → 人写先例卷意图 → bug/需求裁决史 → 欠定决策史）
  ├─ 命中且过采信门限(公式21) → 径直采用，继续产出（不问、不重复写回）
  └─ 缺失 / 记载互斥 / 记载与实机冲突 → ask（呈报-确认式，见 2.6.3）
```

**自然人行为公理（四条）**：①不硬编（不 observe-then-assert、不擅改预期）；②不撂挑子（不自判
env_blocked/escalated 终结——引擎无单方终结权）；③总结呈报差异与**自己的理解**去确认；
④确认后继续产出（目标函数：非产品缺陷一律产出 excel）。

### 2.6.3 题面唯一形态

> 差异呈报（双方引文+各自的锚）＋已检索证明（查过哪些 K_ought 源、各自结果）＋引擎的理解 Z
> ＋「理解正确按 Z 继续 ／ 纠正：___ ／ 确认为产品缺陷走候选单」

挂起/停止是**用户常驻特权**（任何面板可敲），不是引擎推销的选项。修法、诊断、方向选择永不上
面板（实然可导出）。ask 的机械前置=实然手段试尽（重跑/换形态/通道复核先行——ε 消除在冲突呈报
之前，防瞬态差异被当应然冲突）+修法队列为空。

### 2.6.4 收敛律与采信规则

**收敛律（公式20）**：人的每次确认必须以判例键+锚写回 K_ought ⇒ 同键应然问题至多问一次，
ask 数随批次**版本族内**单调不增（跨版本重问=新真意的正确代价，A12）。**自愈的完整定义**：K_is 靠真 PASS 写回自动增长（既有），K_ought 靠
裁决写回自动增长（本节），两层同增使"少问人"与"该问必问"同时成立。

**采信规则（公式21）**：检索命中的应然记载是**证据不是公理**——采信门限：命中记载间无互斥
（单源命中即满足）∧ 与实机不冲突 ∧ 不**改写**意图面（与 D 文本/既有 E 语义相抵才算改写；
从记载**填充**欠定空缺是遵从既有意图投影，不是改写——A11）→ 径直采用；任一不满足 → 呈报。**K_ought 写入权=人源专属**：
引擎自己的推断只能以 uncertain 观察带语境入库，不得写成应然判例（防未经确认的理解自我复制，
与 footprint validity 二态同构）。

### 2.6.5 对抗检查（2026-07-10，12 攻 6 修 6 站——A10/A11 打中的是本节自产公式的 bug；补出的参数即"缺的参数"）

| # | 攻击 | 裁决 | 补出的参数 |
|---|---|---|---|
| A1 | is/ought 分界机器判不动 | 站住（补参数） | **意图触碰面** = {D 语义, E 语义, 交付预期}；P 在正解等价类（§3.3）内变形属实然，触面即应然 |

## 2.7 行动论：判断开放、入库门闭合（2026-07-11 yzg 233 实证补；两轮对抗定形）

### 2.7.1 缺口的发现（实证）

yzg 真机验收：233 案把 listener IP 从物理口移至 vlan100 后清理段未还原 interface 层，
下游 9 案同 IP 超时（R1 6/26 的主因）。两个归因孔的散文**独立且完全正确**（因果链、
受影响案清单、两个修法方向），但认知穿不过三道人写的墙——disposition 枚举无「前驱
污染」词位（正确散文被拍扁成 env_blocked）；固定图上没有「按公共原因组织应对」的
路；引擎无「执行 LLM 提案动作」的出口。若按「加检测器/加枚举条目」应对，则每类
新问题都要人加代码——应对上限=人的预见力上限，是人写规则的自动化，非 agent 自动化。

**理论定位**：本理论此前是知识论完备、行动论缺失——K_is/K_ought 的流动、写回、收敛
均有公理（§2.6），但**行动空间**（能重编/复跑/重排/问询）被隐式锁在人写的 py 层
（图拓扑+工具+枚举）。四层封闭回答了「新知识零代码进入」（判例层），没有对称回答
「新形态的行动如何零代码进入」。知识可自愈生长、行动不能——不对称即缺口。三个
具体冲突：①判定表误分类一次（「失败后的应对组织」会因场景而异，应归 LLM 判断，
却入了确定性流程的格子）；②「LLM 不当胶水」的实证（长上下文执行失控、942 配对）
证明的是**执行序列**不可交 LLM，被过界引用到**判断的组织**；③§11.7 已触到「信息或
**权限**不在引擎侧才 ask」，但未展开「引擎权限内的行动从哪来」。

### 2.7.2 行动论公理

**(22) 行动生成律**：行动空间 A 非常量。A_{t+1} = A_t ∪ {a：a 为 LLM 从原始事实生成
的提案，且 oracle(a)=pass，高危类首执另需人批}。**闭合的不是动作集合，是入库门**
（oracle+人批）；规则不是人写的，是验证出来的。

**(23) 知行对称律**：行动侧与知识侧形态严格对称——

| | 知识侧（§2.6 已立） | 行动侧（本节） |
|---|---|---|
| 开放生成 | 观察/记载（validity 分级入库） | 提案（工作流：动作序列+参数） |
| 闭合的门 | 人源专属写回（A5）+verbatim 门 | oracle 验证+高危首执人批 |
| 判断归属 | 孔（LLM 判层/判形态） | LLM 组合原语与给参 |
| 落账归属 | 引擎（事实流/判例库） | 引擎（执行+效果记账） |
| 收敛键 | 判例键（意图签名×形态×版本族） | 行动判例键（原因签名×形态×版本族） |
| 收敛律 | (20) 同键至多问一次 | 同键免生成免批（检索命中→机械/轻确认执行） |

**(24) 前筛律（效率）**：批级判断的触发条件必须是机械前筛（如 fail 签名最大簇占比
≥θ）。θ 类规则是**资源规则**（同 max_rounds/墙钟/并发上限），不随世界形态增长，
不违背 agent 自动化；**场景规则**（interface 词表、vlan 检测器）才是被本节禁止的
增长方向。

**(25) 写外部世界的双通路**：写共享资源（设备床）只有两条合法通路——机械逆放己方
记账的写操作（框架 IP 契约的推广：写时记账，恢复=逆放账本），或人批准的提案（携
行动判例键，批准即写回，同键复现免批）。LLM 对外部世界**只有触发权与组合权，没有
无验证的生成执行权**（INV-9 的行动论表述）。

## 3. 失效根因空间与分层可导性

### 3.1 根因空间 Ω（五类 + 一）

| # | 根因类 | 实证 |

### 3.2 分层可导性（可导 ≠ 离线可推理）

R\* 不是黑盒，是**分层可查询结构**，每层有原生算子：

| 层 | 原生算子 | 成本 | 可判定性 |
|---|---|---|---|
| 语法层（命令+参数形态） | `?` 逐位置枚举（设备内建语法反射），`^` 定位错误 token | O(1) 零副作用 | **完全可判定** |
| 状态层（对象存在/配置生效） | show 只读 | O(1) 零副作用 | 完全可判定 |
| 行为层（触发→响应语义） | 配置+触发+观测（do-操作） | O(实验) 有副作用 | **实验算子下可导** |
| 值域层（参数边界行为） | 代表元实验（1::1 vs fc00::231） | O(实验×值域) | 边界逐点可导 |

**归因判定树**（fail 后按层下行，每层用本层算子终结，禁止跨层用贵算子）：

```
O ⊭ E
├─ ^ 语法拒绝?    → ? 直查该位置          → 用例错 | 文档错/build差异 | 特性缺失   [O(1) 读表]
├─ 配置生效了吗?  → show 状态对账          → P 未生效(用例错) | 生效了 ↓          [O(1) 对账]
├─ 观测可信吗?    → 第二通道复查            → ε(通道失真,重跑) | 可信 ↓           [O(1) 对账]*
└─ 行为不符      → 分辨实验(换形态/控变量/换值域)                                  [实验,二分假设空间]
                   → 换后通过 = 用例错(形态问题) | 多形态复现 = 缺陷候选坐实
```
\* 第二通道（跳板机直接 dig）当前只在人工诊断路径存在——自动化框架的触发端只有 routera，
引擎内的通道层近似算子是 digest 跨轮对照（重跑消失=瞬态）。对抗检查 A13 如实标注。

**可导性定理（修正版）**：单点观测下 Ω 中多个根因观测等价（044572：①与③在单形态单轮下不可分——attributor 猜③，真值①）；加入分层算子后每个 fail 在**有限次定向操作**内归到唯一根因。语法/状态/通道三层是读表与对账（无需实验设计）；只有行为层需要实验，且实验的角色是**二分假设空间**（五案手动裁决即此算法的人肉执行：历史形态重跑、disable 对照段、换 v6 地址）。

> 推论：形态检验轮（缺陷候选必须换形态复现才坐实）是行为层可导性的结构必需，不是工程偏好。
> 推论：`^`→`?` 自动接线是语法层可导性的算子本体缺口（044572 判死的根因是在可读表的层用了猜测）。

### 3.3 唯一正解（修正版）

- P 不逐字唯一，**正解等价类**唯一：Sound 定义等价类（570 的 monitor default down 与健康检查指不可达 IP 同类；644 的任意合法 ULA 同类）。
- 正解**可查**依赖 K 完备：413 的 lastresort 在手册内（K 足够，检索失败是可达性问题）；493 的聚合语义不在任何文档（②′，只能实验）。
- **K 完备时正解等价类非空且可判定；K 不完备时经有限次实验逼近**——自愈环即逼近过程：

```
K_{t+1} = K_t ∪ Obs_t（观察入库+PASS 升级） \ Poison_t（污染剔除）
R1命中率(t) = f(完备度(K_t), 污染度(K_t), 可达度(K_t))，f 对三参单调
```

## 4. 中心论断：偏差是 K 的状态函数

对抗性反驳（§9 前身）淘汰了"目标函数缺 2/3 需全量 oracle""D 需集中规范化""重编需搜索理论"三个过度理论化的诊断后，存活的中心论断：

> **硬骨头偏差随 K 的健康度涨落；prompt 质量的边际影响远小于 K 状态。**

## 5. K 状态机

### 5.1 状态空间

**σ(k) = (validity, reachability, anchor)**，两维正交 + 一锚。

**有效性 validity（条目内容与 R\* 的关系）——七态：**

| 态 | 定义 | 实证 | 检测信号 |
|---|---|---|---|
| `absent` | 所需语义不在 K | 493 聚合语义、570 本地域名语义 | 无直接信号；代理：检索零命中日志 ∧ 换形态后 PASS |
| `uncertain` | 观察级未实证 | P1 入库 6 条 fail 轮观察 | validity 字段（已实现） |
| `verified` | 实证带 PASS 锚 | device_verified 写回 | 锚完整：autoid ∧ run_ts ∧ **build** |
| `conditional` | 多语境并存（verified 精化；**知识的本态非坏态**） | disable→DOWN（纯 service pool）vs 恒 UP（挂 CNAME 池）；obsA/obsB | 同主题互异 observed_under ≥2（观察组计数，已实现） |
| `contested` | 血统间断言冲突待分辨 | 手册 re-query vs 某形态实测 ANSWER:0 | manual↔device 同主题谓词冲突；常为 conditional 未分化前身 |
| `stale` | 曾真、R\* 已漂移 | 644 历史卷（当年 1::1 PASS，本 build 不响应） | **无自发信号（最危险）**；唯一可靠信号=锚差：条目 build ≠ 当前 build |
| `poisoned` | 从未真、带假印记 | 首批 608/570 毒卷；644 弱化卷（写回即毒） | 入库后：意图↔断言方向体检；入库前：单调性/投影门 |

> verified 不是终态，是需要维护的态——stale 与 poisoned 都伪装成 verified，且 stale 的形成完全无声。
> 这是"K 运维"区别于"K 积累"的本质：积累管入库，运维管**印记的持续有效性**。

**可达性 reachability（条目与消费者的关系）——四态：**

| 态 | 定义 | 实证 | 信号 |
|---|---|---|---|
| `reachable` | 正常命中 | — | — |
| `latent` | 在库但时窗错位 | 570 R2 正解批末写回、608 R3 检索不到（即时写回已修） | 写回时间戳 > 消费时间窗 |
| `lexically-unreachable` | 词汇鸿沟 | 413"回退池"↔lastresort | 检索零命中 ∧ 事后条目存在；该信号对本身可入库为词汇映射条目 |
| `suppressed` | 设计性不可达 | uncertain 不进 allowlist | 显式规则 |

正交性：413 = (verified, lexically-unreachable)；644 弱化卷 = (poisoned, reachable)。**单维视图必漏。**

### 5.3 具体状态机：append-only 事实 + 派生谓词（可实现规格）

**设计原则（回应 C4 复杂度挑战）**：状态**不存储枚举、不写迁移代码**——条目上只追加事实字段，
状态是查询时的派生视图。迁移 = 事实追加，状态自动跟新。若实现中出现按状态组合分支的 if 树，即违背本规格。

**条目 schema（在现有 RawFact/节点条目上的增量，✅=已实现字段）**：

```jsonc
{
  "fact_key": "...",                    // ✅ 身份
  "validity": "verified|uncertain",     // ✅ 主态(唯一存储的枚举,二值)
  "observed_under": "……",              // ✅ 观测语境(自由文本短句)
  "conflicts_with": "fact_key",         // ✅ 冲突声明(人工/机器均可追加)
  "evidence": {...},                    // ✅ 血统载体(source_file→manual / device_evidence→device)
  "anchor": {                           // ⚠ 增量:三元锚
    "autoid": "…", "run_ts": 0.0,       // ✅ 已在 device_evidence
    "build": "10.5.0.583"               // ✅ 已落(2026-07-09)——链路 verified_runs.jsonl(digest 记提交 build)
                                        //    → device_run_ref 透传(writeback/closing) → 条目 evidence.device_run.build;
                                        //    stale 派生判定(锚差)待建
  },
  // build 的三类来源(2026-07-09 用户补充,各服务不同的锚语义):
  //  ① show version(设备自述)——device 血统条目的实证 build:写回/digest 时采集
  //  ② 框架 report 目录名(InfosecOS_Beta_…_10_5_0_568)——跑批时零成本旁路采集
  //  ③ 缺陷库记录(报于 vX/修复于 vY/合入 vZ)——known_issue 条目的锚不是单点而是
  //     **版本区间**:stale 判定 = 当前 build 与 [报出,修复) 区间的关系(044572 的
  //     attributor 曾做此推理但未结构化:「r44409 修复于中信分支,被测设备未包含」)
  "fix_version": "…",                   // known_issue 专属:修复/合入版本(来源③)
  "quarantine": {"reason": "…", "ts": 0} // ❌ 缺——poisoned 处置=隔离标注(不删,留案底+引用清算)
}
```

**派生谓词（全部 O(条目) 可计算，查询时求值）**：

```
lineage(k)      ≝ device  if k.anchor.autoid≠∅  else manual        （血统）
stale?(k)       ≝ validity=verified ∧ lineage=device ∧ anchor.build ≠ current_build
conditional(k)  ≝ ∃k' ∈ 同节点: 主题(k')=主题(k) ∧ observed_under 互异非空   （观察组计数,已实现）
contested(k)    ≝ conflicts_with ≠ ∅ ∨ ∃k' 同主题异血统且谓词相反
poisoned(k)     ≝ quarantine ≠ ∅        （由体检/单调性违例回溯**追加**,非实时判定）
absent(q)       ≝ 检索(q) = ∅           （查询的属性,非条目属性——只有代理信号,见 C2）
latent(k, t)    ≝ k.write_ts > 消费窗(t) （已被即时写回消灭为 0 时窗）
lex_unreach(q,k)≝ 检索(q)=∅ ∧ k 事后被证含正解  （事后审计谓词,产出词汇映射条目）
```

**迁移逻辑 = 事实追加表（谁、在什么事件、追加什么字段、渲染副作用）**：

| 事件 | 执行者 | 追加的事实 | 派生态变化 | 渲染副作用 |
|---|---|---|---|---|
| fail/escalated 轮观察 | closing._ingest（✅） | validity=uncertain + observed_under + anchor.autoid | absent→uncertain | `[uncertain\|语境:…]` |
| 同 fact_key PASS 实证 | merger 升级分支（✅） | validity→verified + 新 evidence | uncertain→verified | 标注消失/更新 |
| PASS 写回 | writeback_one（✅） | 条目 + anchor(autoid,run_ts **+build**❌) | →verified(锚全) | — |
| 第二语境观察 | closing._ingest（✅） | 新条目 + 互异 observed_under | →conditional（计数派生） | 观察组组头 |
| 冲突发现 | 策展/对账器 | conflicts_with | →contested | ⚠条件冲突横幅（✅） |
| 分辨实验裁决 | attributor/人工 | 胜者 validity=verified + 语境钉死；败者 quarantine 或改写语境 | contested→conditional/resolved | 横幅解除 |
| **build 变更** | 跑批入口探测（❌） | 无追加——current_build 变化使 stale?(k) 谓词翻真 | verified→stale?（批量派生） | `[stale?\|实证于X,当前Y]` |
| stale 复验 | 懒惰:消费前重验（❌） | 通过→anchor.build 刷新；不过→quarantine | stale?→verified / poisoned | 标注更新 |
| 体检命中 | 方向体检扫描（⚠手动预演） | quarantine{reason:方向翻转} | →poisoned | 条目不再渲染,案底可查 |
| 单调性违例 | emit/merged 门（❌缺口B） | 拒绝产出（入库前拦截,无条目产生） | —（防 poisoned 出生） | worker 收到违例说明 |
| 词汇鸿沟事后审计 | 归因复盘 | 新条目:意图词→实现词映射(自身带锚) | lex_unreach→reachable | 检索命中 |

**消费端契约**（信号如何改变使用姿势——唯一允许的"按状态分支"，且全部是读侧权重）：

```
verified ∧ ¬stale?   → 直接引用
uncertain            → 可引用,标注不确定;设计实验时优先当仲裁对象
conditional          → 对照自身配置形态选语境分支;不匹配任何语境→当 uncertain 用
contested            → 禁止单边引用;触发分辨实验或如实报告矛盾
stale?               → 引用前复验(语法层 ? 对账便宜先行,行为层懒惰重验)
poisoned(quarantine) → 不渲染;归因/复盘可查案底(负信息量,严禁进 brief)
```

### 5.4 统一原语：K 运维 = 锚差监控

三大运维职能还原为同一操作——**比较条目的锚与现实**：

- **污染检测** = 有效性锚差：断言方向 vs 意图方向（poisoned）；build 锚 vs 当前 build（stale）
- **对账** = 血统间锚差：manual 条目 vs device 观察的同主题谓词冲突（contested 发现）；语法类条目 vs 设备 `?` 自反射（语法层批量对账——文档错误从"事故现场发现"变"库存盘点发现"）
- **流动** = 可达性锚差：写回时间 vs 消费时窗（latent）；意图词 vs 索引词（lexical）

**信号传递模式**（沿用 P1 已验证通道）：状态标注在条目、渲染层带标——`[uncertain|语境:…]` 是首例，同通道容纳 `[stale?|实证于build568,当前583]`、`[contested|与手册冲突,实验可仲裁]`。**信号改变消费姿势，不改变可见性**——与"uncertain 不冒充 verified 但照样入库"同一哲学。

### 5.5 第二状态机（case 台账）的同构纪律与 oracle 残差公理（2026-07-10 yzg 实证补）

case 台账与 K 库是并列的两个状态机，但截至 yzg 批，**K 库的三条纪律没有一条被搬到 case 面**——
yzg 终验吞 fail（3 案整卷 fail 被 pass 终态锁静默丢弃、照常写回先例库）暴露的正是这组缺失：

| K 库已有纪律 | case 台账对应缺失 | yzg 实证 |
|---|---|---|
| 条目带 (validity, observed_under, anchor)——结论永远带语境 | `passed` 是无语境布尔（∃-pass 冒充交付-pass） | 子集 pass 锁死后，整卷 fail 无处落笔 |
| 矛盾观察共存（观察组）+ 实验仲裁，不销毁 | 矛盾裁决被恒等补全吞掉（δ(passed, oracle_fail) 未定义→保持原状） | frozen 信号发了、无人消费，1 秒后写回 |
| 信号闭集从迁移表对抗推导（C1-C10 挑战过） | case 迁移表从未做对抗性补全，「oracle 反证终态」事件类未被枚举 | 无信号名、无 ask 边、无报告计数 |

由此固化四条公理级纪律（编号续 §6 公式表）：

- **语境锚**：case 终态必须携带取得语境（passed@subset ≠ passed@full-volume）；交付判据只认
  ctx_delivery 下的 pass。
- **oracle 残差公理**：每条上机裁决必须恰好落入 ①改变状态 / ②显式核对一致 / ③产生显式残差
  （信号/升级/ask_user）之一，**禁止无痕吸收**。部分函数 δ 的未定义点不得以恒等补全——
  「事件被正确处理且状态不变」与「事件被丢弃」观测不可分辨，是最危险的静默失效形态。
- **全射对账**：digest→台账按 **dom(V)**（本轮跑了谁）正向遍历，不按 L⁻¹(failed)（台账里谁挂着）
  反向索引。两个集合在子集轮巧合相等（掩盖了缺陷两个回合），终验轮不相交（3 fail 落空）。
  问题#7（子集轮 frozen 失活，路径键错位）与 #10（终验吞 fail，状态键错位）与更早的
  「瞬态护栏 dead code」是这同一条缺失公理的三次独立显形。
- **审计器权威**：审计一条假设的机制必须比该假设权威更高。整卷终验是「子集≡整卷」独立性
  假设的审计器，却被置于该假设之下（假设先锁终态，审计结果反而进不来）——审计沦为仪式。

**问询纪律的对称化**：现行 ask_user 两条边（编写期欠定 / 终局升级转述）都处理*认知不确定性*，
且都长在合法状态迁移下游。缺第三条边——**矛盾即问**：oracle 反证已锁结论时，
只有用户有权裁决（如实报告红线）；题面形态后经 §2.6 收敛为呈报-确认式
（不再是「降级/重排/接受单跑」选择题——那是三稿前的形态）。
进不了状态机的事实永远变不成问题，故此边的前提是 oracle 残差公理先成立。

## 6. 理论公式汇总

```
(1) Sound(C)      ≝ (P ⊩ D) ∧ (E ≅ proj(D)) ∧ (R*(P) ⊨ E)
(2) Verify(C)      = [O ⊨ E],  O = Exec(P) + ε          （只测合取③）
(3) Monotonic      ≝ ObsCmds(new) ⊇ ObsCmds(old) ∧ Asserts(new) ⊇ σ(Asserts(old))
                                                        （合取②在压力路径的可判定投影）
(4) Ω              = {D错, P错, E错, K错, K缺, 缺陷, ε, 欠定}
(5) 可导性         : ∀fail, 沿分层判定树在有限次原生算子调用内 ω 唯一
                     语法/状态/通道层 O(1)；行为层 = 实验二分
(6) σ(k)           = (validity, reachability, anchor),  anchor = (build, run_ts, lineage)
(7) 检测原语       : Δ(k) = dist(anchor(k), reality)    （锚差；所有信号的母式）
(8) K 演化         : K_{t+1} = K_t ∪ Obs_t \ Poison_t
(9) 中心论断       : R1命中率 = f(完备度, 污染度, 可达度)，f 单调；prompt 为低阶项
(10) 正解          : K 完备 ⇒ Sound 等价类非空可判定；K 不完备 ⇒ 经实验有限逼近
(11) 信息量分解    : I(C) = (I_G, I_E, I_V)，⊔ 不可通约（不可补偿+用途不同）
                     I(a) = -log₂ P(O⊨a | 行为无关假设)；恒真 ⇒ I=0
(12) 覆盖判据      : dim_V(C) ≥ dim_V(D)   （代理: genuine_v_count vs step_intents 行为断言数）
(13) 守恒律        : 重编 ΔI_V ≥ 0          （单调性检查的信息论内容；644 违例=dim_V 3→2）
(14) K 净信息量    : 运维目标 = 保持 Σ_k I(k) > 0（stale/poisoned 条目 I<0，主动误导）
(15) pass 语境化   : Pass(c, ctx)；交付判据 = Pass(c, ctx_delivery)；
                     ∃ctx.Pass ⇒ Pass(ctx_delivery) 仅当 case 独立性成立（保存族反例）
(16) oracle 残差   : ∀c ∈ dom(V): consume(V(c)) ∈ {迁移, 显式一致, 显式残差}——禁止第四种
(17) 全射对账      : 对账遍历 dom(V)（本轮裁决集），非 L⁻¹(failed)（台账反查）
(18) 审计器权威    : authority(审计机制) > authority(被审计假设)，否则审计为仪式
(19) Ask 定理      : ask ⟺ 决策依赖应然 ∧ K_ought 检索余项非空
                     实然=可由{读,查,实验,算}决定(永不问)；应然权威=意图所有者
(20) 收敛律        : 确认必写回 K_ought(判例键+锚) ⇒ 同键至多问一次，ask 单调不增
                     自愈 ≝ K_is(真PASS写回) 与 K_ought(裁决写回) 双层自动增长
(21) 采信规则      : 采用(记载) ⟺ 命中记载间无互斥 ∧ 不与实机冲突 ∧ 不改写意图面
                     改写=与 D 文本/既有 E 语义相抵；填充欠定空缺≠改写(A11)
                     K_ought 写入权=人源专属；引擎推断只入 uncertain 观察
(22) 行动生成律    : A_{t+1} = A_t ∪ {a: LLM 提案 ∧ oracle(a)=pass ∧ [高危→人批]}
                     闭合的不是动作集合是入库门；规则是验证出来的不是人写的
(23) 知行对称律    : 行动侧(提案开放/oracle+人批闭门/组合归LLM/落账归引擎)
                     ≅ 知识侧(观察开放/人源写回闭门/判断归孔/落账归引擎)
(24) 前筛律        : 批级判断触发=机械前筛(签名簇占比≥θ)；θ 是资源规则非场景规则
(25) 外界写双通路  : 写共享资源 ⟺ 机械逆放己方记账 ∨ 人批提案(携行动判例键,同键免批)
                     LLM 只有触发权与组合权,无无验证的生成执行权(INV-9 行动论表述)
```

## 9. 对理论的挑战与未决问题（诚实清单）

- **C1 合取②全量不可判定**：忠实投影的一般判定=语义理解。记录类型 gap 抓 644 靠"AAAA"词面出现——纯巧合，脑图写"v6 地址"即失效。唯一全量实现是独立 LLM 判定器，而其判别力从未被正确度量（grade 之死的 942 对配对拿上机结果错标忠实性）。**当前立场：单调性守压力路径；R1 直接歪写 E 的威胁现代系统下实证为 0，列为待观察；DS-2 是任何未来候选的准入门。**
- **C2 absent 原理上不可观测**："不知道自己不知道"只有代理信号（检索零命中 ∧ 事后正解在 R\*）。自愈环把 absent 的暴露成本压到"一次 fail"，但压不到零。②′ 类根因的首次遭遇必然付一轮实验学费——这是知识论边界，不是工程缺陷。
- **C3 stale 扫描的成本与观察者效应**：build 锚只能缩小嫌疑集（build 变了 ≠ 语义变了），确认 stale 仍需重验，全库重验烧设备。可行策略只有懒惰验证（消费前验）+ 分层优先（语法层 `?` 对账便宜先做，行为层抽样）。锚差是必要信号非充分判据。
- **C4 状态机自身的复杂度风险**：七态×四可达=组合膨胀？辩护：状态是**条目上的标注**不是流程分支——消费端只读标注调权重，生产端只有锚差比较器。若实现中出现按状态组合写 if 的代码，即违背本设计。
- **C5 锚的可信性递归**：锚差监控的前提是锚不可伪造。run_ts/verified_runs.jsonl 在 runtime/（agent 沙箱黑名单）——锚可信度依赖沙箱完整性，沙箱是本理论的信任根。
- **C6 中心论断可证伪性**：若未来批次 K 健康而硬骨头依旧（DS-4 监测），§4 被打——届时候选解释按序重启：合取②首轮失真（C1 待观察项转正）、行为层实验设计不足、D 解码方差。**理论的死法已预注册。**
- **C7 词汇映射条目的递归性**：意图词↔实现词映射自身是 K 条目，同样会 stale/poisoned（产品改名）。量小，接受递归一层。
- **C9 manual 血统无 stale 锚**（对抗检查 A4）：手册无 build 粒度（manual_10.5 覆盖整个 10.5 家族），跨 build 行为差异（568→583）手册不区分——manual 条目的时效性**不可锚定**，其 stale 只能经 device 血统观察冲突（contested 路径）间接暴露。stale?(k) 谓词只对 device 血统有效是设计而非疏漏。
- **C10 poisoned 的引用传染不可查**（对抗检查 A6）：条目被 quarantine 后，引用过它的下游产物（据它编的先例卷、以它为 evidence 的观察）无引用图可循——644 弱化卷的回滚靠人工记忆。自动清算需要引用链索引（provenance 的 ref 链是雏形，未建反向索引）。当前立场：quarantine 记案底 + 富集扫描（fail 后强行 PASS 血统毒率 20%）近似定位传染面。
- **C11 case 状态机未经对抗性补全（2026-07-10 yzg 实证，已定性）**：信号闭集/迁移表的对抗
  推导只做了 K 状态机；case 台账的迁移表存在未定义点（δ(passed, oracle_fail)），实现以恒等
  补全——三次独立显形（瞬态护栏 dead code / #7 冻结失活 / #10 终验吞 fail）同根于
  「digest→台账无必达契约」。§5.5 四公理是补全方案；落地前该缺口视为开放。
- **C8 conditional 的语境不可枚举**：observed_under 是自由文本，语境空间不闭合；观察组计数是弱信号（≥2 才触发，首个语境相关观察无从标记）。缓解：conditional 是默认怀疑姿势（任何 behavior 观察都可能语境相关），渲染层"多语境观察，对照你的配置形态取用"即此姿势的固化——**宁可让消费者多一分怀疑，不给机器加语境枚举的假精确**。
- **C12 行动论的冷启动与视野边界（§2.7 两轮对抗的站住项）**：行动判例库空载期纯付出
  （蒸馏/检索 miss/人批频繁），有界性靠"公共原因形态量级 ~10"的假设——若形态长尾比预估
  重，收敛慢于 3-5 批（E6）；批级诊断的视野=本批事实+检索半径，跨批因果链依赖检索命中
  而非结构保证（A4）。两者均预注册为 DS-4 型监测项：库命中率与人批频率若不随批次单调
