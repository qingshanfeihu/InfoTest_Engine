## 1. 分层状态空间与配置依赖图（公式 (28)）

**手册实证**：26 章严格按功能域组织，数据面对象可映射 OSI 层——链路聚合(L2)、
基本/高级系统配置含 interface/vlan/ip/route(L2/L3)、HA/集群/SSI(L2/L3 冗余)、
SLB/QoS/LLB(L3/L4)、SSL/HTTP/DNS缓存/GSLB/安全/DPI(L5-L7)；另有管理面 M（日志/
SNMP/WebUI/管理工具——不参与承载链但参与污染与复位，R6-G8）。

**(28) 分层状态空间**：S = (S₂, S₃, S₄, S₅₋₇, S_M)，配置对象间存在**依赖图 G**：
节点=配置对象，边=命令签名中的跨对象引用，节点携层标注。

依赖边在手册命令签名里**明文存在**（参数占位符即边的类型声明）：
`ip address <interface_name> …`(L3→L2)、`slb virtual tcp <vs> <vip> <vport>`(L4→L3)、
`sdns listener <ip_address>`(L7→L3)、`ha group fip <gid> <floating_ip>
[interface_name]`(L3虚→L2)、`slb group/policy → virtual/real`(L7策略→L4成员)。
手册说明文字还记载功能级依赖（「未配置SDNS服务监听，则SDNS功能无法正常使用以及
SDNS全域名功能不生效」——listener 是 sdns 与 fulldns 的公共祖先）。

**类型图与实例图分立（R6-G1）**：手册解析产出**类型图**（边模板：sdns listener

## 2. 四算子代数与 clear 作用域格（公式 (29)）

**手册形态学统计**（10.5 CLI 29 分册）：no 命令 633 条、show 命令 899 条、clear
命令 605 条；61 个顶层命令名中 55 个有对应 show 投影。

**(29) 四算子及其律**：
```
config X   : 状态变换 s → s'
no X       : 配置面逆元 —— no X ∘ X = id  (限配置面,见 R6-G6 边界)
show X     : 只读观测投影 π_X(不改 S;61 顶层中 55 个有投影,6 个在投影核外)
clear X    : 子空间幂等复位 —— clear X ∘ clear X = clear X
```
**clear 作用域格**（包含链）：`clear <feature>`（单功能域：slb 92/statistics 76/
sdns 71/http 40…）⊂ `clear config primary/secondary/file`（配置层）⊂
`clear config all` ⊂ `clear config factorydefault`（出厂）。

**逆元律的诚实边界（R6-G6）**：no X 恢复的是**配置面**，不恢复**状态面**（会话表/
统计计数/磁盘持久文件/对端已同步内容）——「持久化家族」的数学定义即**状态面写**
（配置面逆元对其无效的算子），不再需要枚举通道：save/synconfig/segment 文件全部
落此定义。状态面恢复归 infra 域（快照/T4/床账）。

## 3. 承载合取律（公式 (30)）

数据面包流：触发端 →(L2 邻接/vlan)→(L3 可达/ARP)→(L4 监听)→(L5-7 服务逻辑)→ 返回。

**(30) 承载合取律**：Avail(o@L7) = conf(o) ∧ listen(anc₄) ∧ reach(anc₃) ∧
adjacency(anc₂)——功能可用性是其在实例图上全部祖先的合取。**一个 L7 断言
（check_point）是对整条承载链的合取断言**，与三合取定理（§2 横向）同构，方向为纵。
推论：L7 fail 有跨层候选根因集=其祖先链——归因难度的数学本质。

**冗余修正（R6-G2）**：HA 双机、多 listener、lastresort 等冗余结构下，合取在冗余组
处变析取（k-of-n）——精确形态是 and/or 图。第一版取合取近似（过报不漏报），呈报时
声明近似；冗余组建模待实例图成熟后补。

## 4. 传播定理与复位差集定理（公式 (31)(32)）

**(31) 传播定理**：写 Lk 层对象 o 的影响集 = o 在实例图上的**上闭包**（所有经 o
承载的高层对象）；**扇出反比于层**（一个 L2 接口承载 N 个 L3 地址×每地址 M 个
L4/L7 服务）——越底层的写，爆炸半径越大。
实证：233 改 L2（vlan/port2）→ 34.70@L3 断 → 9 个 L7 案全灭，40/73=54% 的 fail
是一次 L2 写的上闭包（R5 定量）。

**(32) 复位差集定理**：污染逃逸面 = 用例可写空间 W − per-case 复位空间 R。框架
per-case 清理=clear slb/sdns（L4/L7 功能域），L2/L3 对象（interface/vlan/route）
属系统配置章、不在任何功能 clear 作用域——**差集恰好集中在扇出最大的底层**。
「共享床上底层写最危险」不是经验而是 (31)×(32) 的交：逃逸面与爆炸半径在低层重合。

## 5. 观测结构与归因距离

- **投影核**：6 个顶层对象无 show 投影+跨设备盲区（触发端的 L2 邻接形态——tagged/
  untagged——不在被测设备的任何投影里，233 首轮实证）——不可观测分量的污染无法用
  快照 diff 捕获（infra 床态快照的能力上限，此处给出其数学根源）。
- **归因距离**：断言几乎都打在 L7 回显，根因可在 L2-L7 任一层；归因成本 ∼
  断言层与根因层的**链上距离**。推论：正确的归因结构=**沿承载链下降**（L7 断言
  fail → L4 在听吗 → L3 可达吗 → L2 邻接对吗），每层用该层 show 投影，是机械探针
  序列——attributor 现有「版本层下降」之外缺的正是 OSI 链下降。

## 6. 版本参数化（公式 (33)）

**(33)** S 的分量集是 build 参数化的：S(build)。fulldns∉S₅₋₇(10.5.0.585) 而
10.5 专属 CLI 手册与设备行为一致（fulldns 事件三级反转的终局定性）——「命令 ∈
S(build) 的命令集」是可机械判定的（版本专属 CLI 手册命令集成员判定），emit 期即可
拦「测本版本不存在的功能」，无需烧三轮设备。

## 10. 时间轴：候选公式 (34)（2026-07-12 缓立——数据不支持即停的示范案例）

**候选命题**：状态空间分**配置面**（本案算子写之间静止，断言时不变）与**状态面**
（自治演化：TTL 过期/health check 定时器/会话老化/统计累积/RR 轮转态）；

```
(34·候选) 对状态面的断言是时间索引断言 O ⊨_t E ——
          合法形态 = 等待-验证对 ∨ 容差窗,窗宽从该对象的时间常数导出
          (时间常数在手册签名的 interval/timeout/ttl/expire 参数里,类型级可解析)
```

**为什么只是候选（检验一，`RESEARCH_theory_data_checks.md §1`）**：yzg 数据不
支持——①sleep 的 92% 时变邻近富集是块大小混杂+单签名饱和假象（分层后归零），
且 13 个 sleep 全是「等配置生效」型；②8 个翻转实例中 2 例翻在配置面断言（直接
反例），翻转主导解释是床污染（infra T1），0 例归因自治演化。唯一强定性支持
（zhaiyq 22/50 persistence TTL 时序翻转）原始卷不可得。

**域外背书充分**（文献调研二）：safety/liveness 二分（TLA+）、flaky 第一根因
async wait 45%（FSE'14）、QuickLTL（PLDI'22，含警告：无界 eventually 在有限
trace 退化恒真——与 not_found 恒真同族）、RFC 6412 收敛时间标准化、Awaitility。
即：**理论方向有工业/学术三层背书，但「它是本域主要失效模式」的域内证据未至**
——yzg（sdns listener 配置域）天然偏配置面，没给该命题施展对象。

**处置**：①不立门不进 emit；②手册时变签名表（202 条）作为文法层数据随类型图
解析（DESIGN §15 S1）捎带落盘——零代码增长，数据先备好；③转正前置条件=一个
persistence/health-check 域批次的原始数据支持预言②；④sleep 治理维持队列末位。

**多写者边界（S8，检验一附带修正，即刻生效）**：本文档「配置面在算子写之间
静止」的「算子」限定为**本案算子**——共享床是多写者，他案写造成的配置面变化
归 infra T1 域（床账/hermetic 辖），既不是 (34) 的反例，也不在其辖区。单案
建模视角的这条边界此前未声明，是检验一 2 例「配置面翻转反例」的正确归档位。
