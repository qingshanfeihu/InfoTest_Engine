### 0.1 原始对象

设备是不可直视的机器，能做的只有发输入、收输出：

```
M = (S, U, Y, δ, λ)   Mealy 机
S 状态(不可直读)  U 输入(CLI 命令与网络包同类——都是输入)
Y 输出(回显与响应包)  δ:S×U→S 转移  λ:S×U→Y 输出
```

### 0.2 算子分类的相对性（四算子降为定理）

config/no/show/clear 不是原始概念，是 U 按 (δ,λ) 性质的等价类，且**分类必须
相对于状态分解定义**：dig 对配置分量是纯读（δ 恒等），对运行分量（统计/会话表）
是写。「激励 vs 观测分不清」的病根即在此——分类不是输入的内在属性，是输入
相对于状态子空间的属性。故必须先推导状态分解。

### 0.3 状态空间的三轴分解（面/层/域首次获得数学定义）

分解的唯一合法依据是**动力学性质**（手册章节是文档组织，是派生物非依据）。
三轴是三个**独立定义**的划分（每个分量同时携三标签；「正交」指定义相互独立，
非统计独立——相关性本身是内容，见轴三）：

- **轴一（稳定子分解）→「面」**：按「谁使它变化」——x_cfg 只被操作者输入改变
  （配置面）/x_run 被流量与时间自发改变（运行面）/x_persist 被显式持久化操作
  改变、跨重启存活（持久面）。G6 的二分是其粗化。
- **轴二（影响 DAG 分级）→「层」**：分量间影响关系生成 DAG；**OSI 层不是公理，
  是「该 DAG 近似分级」的可检验经验命题**。(31) 上闭包=DAG 可达性。
  **可复现原料**：手册 29 分册机械解析出 **3572 条命令签名**（覆盖 99.64%，与提交
  `command_inventory_10.5.json` 逐位一致）——这是类型图的输入。**依赖边 DAG 本身
  （定义-引用配对+层标注）尚未构建**，作为「可推导待建数据·类型图」列入 §7；构建后
  跨层边的低→高定向性即可机械核。结构性观察保留（供构建时验证）：反向边预期集中在
  「低层健康/过滤行为引用 L3 字面值」（bond health / bridge apprule 族），非命名对象引用。
- **轴三（交换中心分解）→「域」**：子空间独立 ⟺ 各自输入可交换（交换子为零）；
  **功能域=极大可交换分解的块**。硬推论：**跨域干扰 ⟺ 两块共享底层分量**（交换子
  非零；⇐ 方向为**必要非充分**——共享底层是干扰的必要条件，充分化需补「受害方观测
  π 依赖该分量所在面」）。这是**可机械计算的互扰预测器**：写实体 ∩ 触碰实体于底层/
  持久分量（承载图闭包版——655173 的 IPv6 受害与 vlan100 污染不同名但同承载 port2）；
  判据本体已在库（diagnose 的 s₀ 配对）。查全/精确率的标定数字来自单 yzg 域、脚本已丢
  不可复算，故**不作定量宣称**，升定理需多污染源受控实验（预注册）。
- **域的纤维性**（考古提炼保留）：脑图一级节点=手册功能域章=轴三块=footprint
  前缀组织——四层表示共享同一分区单元；意图低秩、写回增益按域、跨脑图零重合/
  域内骨架部分重合皆由此结构必然（定量快照见 K §2.11，已降为解释性、不作判据）。

**持久面细分（2026-07-14 用户域教学定形；写保存族 σ 链断点的模型根）**：轴一的
「持久面」此前混淆两个子面——**制品子面**（write mem/file/all/net 的产物：本地文件/
远端备份，跨重启存活但**启动惰性**，不参与开机配置构建）与**启动子面**（开机时构建
运行配置的来源）。本产品：**运行配置为内存易失、不落盘；write 族全部是导出动作**
（手册措辞即「备份到…文件」），无一创建开机持久性——重启=内存释放，配置必然消失，
**重启后配置仍在=产品缺陷**。**模型效度声明（2026-07-14 三稿,对抗评审 T2 定形）**：配置面模型（运行=内存
易失/write 族=导出/无启动面）是**用户域教学 K_ought 判例+可错声明**,非无条件
事实——版本族锚在案、设备验证状态=**未验**,且仓内存在已登记张力:手册 HA
bootup synconfig 节明文「启动时同步通过 write memory 保存的配置」（app_21
§synconfig）——模型下发必须携「HA bootup 模式除外」声明;唯一证伪路径=授权床
一次性重启实验（登记,(25) 人批高危）。把模型当无条件文法事实消费=脑图圣化的
同构复刻（位移一层）。

三推论（①③经用户二次教学+对抗评审 T1 修正;「模型担保等价」表述**撤回**）:
①此类意图的证伪观测=**运行面清除后，配置不经显式导入而再现**。重启与 clear
同为「清除运行面」的实现,**在配置面模型成立的世界里**对该命题等价——故禁令床
上的合法编译=改过程「重启→clear 验证」,携**模型条件等价**论证与差异声明
（启动通道缺口:若缺陷形态是「write 暗中产生启动时被消费的持久物」,clear 形
测不到、唯重启形能——该缺口必须进面板差异声明与交付报告,不得以「担保等价」
开路）。仍然非法的替代:回放（导入方向相反）与查制品内容（导出本就全量,测反）。
**等价判据四条**:同面清除（作用域=σ 差集∪π 支撑分量,非全面——全面清除即
A 层禁令）∧ 证伪观测不变 ∧ 不引入反向（导入）操作 ∧ **对被测机制保持敏感**
（删掉被测写步应改变判决;clear 形对 write 步在模型违背世界不敏感,即此条的
实弹反例——判据②在模型相对改述上求值会失明,第④条是其对偶护栏）。
②τ/s₀ 的「持久面写」判据实际检测**制品子面**残留（文件跨 run 存活污染后继）
——判据仍立，但不得由此推出「配置被持久化」；run13「恢复泄漏内部对象」与
2026-07-14 前「设备把 listener 持久化了」两个叙事均因两子面混用而失真。
③替代评估的两个对偶失效模式（同日双实证）：**发明伪替代**（查载荷——未过
等价判据）与**漏掉真替代**（跳过/挂起——未在模型下穷举同面实现）。「无有效
替代」的结论必须携带模型下的穷举论证,**枚举基**=clear 格节点 ∪ inverse_forms
逆元 ∪ 其组合（评审 T18:无枚举基的穷举义务不可证伪）。

### 0.5 隐藏自由度 h：欠定与 flaky 的统一

真实输出 y=λ(s,u,**h**)，h=既不能制备也不能观测的自由度（RR 轮转指针/调度/
时钟/并发/共享床上他人的写）。分类按 **h 的位置**：

```
h 在 λ 内   → 欠定:R 真值本质依赖 h → 对策=R 边缘化为 h 不变式(区间/集合)
h 在 s₀     → 床污染:再分【己方写账(可机械逆放)|他人写(共享床治理)】(B6 子轴)
h 在 π 通道 → 取证/传输噪声 → 复跑/通道修复
无 h        → 确定性错误 → 出本框架辖区,归 Ω 判定树 + is/ought 仲裁
```

**机理推论**：「复跑可救」是 h 位置的推论——π 噪声复跑=h 重采样可救；s₀ 污染
复跑 h 冻结在脏床不可救（除非清场）。**欠定/缺陷分界**：单次观测依赖 h=欠定；
h 边缘化后统计量仍偏离=缺陷候选（wrr 配比异常的正确归档）。**h 位置分类是可判据的归因升级**：
同一签名跨 run 换标签的「误归漂移」现象在盘上直接可见（655154 三 run 三标签、668030 单 run
五标签轨迹均可追，跨 yzg/dongkl/selfheal 多脑图），其结构解释=env_blocked 词位表达不了
「批内前驱污染」、瞬态与床污染共用出口——**h 位置轴（λ / s₀ / π / 无 h）把它们区分开，
同症异判在 h 轴上结构性不可能**。跨批重标的精确计数依赖已丢脚本、不作定量宣称；现象与其
结构解释是可复核的定性事实。
**可判性三档（诚实）**：h-in-λ 与无 h 离线即判；s₀/π/真环境故障**单案观测等价**
（B1），判定成本 O(批)——批级算子（同批 pass 对账/床账/跨轮对照）大半已在，
这正是「单案填词表必然误归」的结构原因。边界 B1-B7 详见
`RESEARCH_theory_data_checks.md §5.5.3`。

**h-不变式的断言形态要求（2026-07-15 dongkl 593516 实弹补；DESIGN §18.15 收口）**：对策
「R 边缘化为 h 不变式」有一条**形态**要求实现必须强制——断言须是 h-不变量的**函数**、不能是
h-**样本**的函数。RR 轮转（h=轮转指针）下，「某成员命中 N 次」是 h-样本读数（同一配置态、
不同采样窗给不同计数）→ 断言真值依赖 h → flaky（同一 s 可 pass 可 fail）→ **不构成 oracle**。
正解形态=绑 h-不变式：轮转成员**集合**=期望成员集（集合相等），或命中**分布**落某区间——都是
h 边缘化后的量。**活违反**：593516 用 dig-命中计数（h-样本）断言 RR，子集复测采样够散即 pass
并 writeback verified——采样噪声被写成第二权威判例源（§2.9.4 (45) 防自指的反面）。外部同构：
NAPALM `_mode: strict` 集合相等、AutoSpec canonical-flow 限制——「采样敏感断言无效」在成熟框架
是现成算子（K 文档 §2.13）。**判据（emit 期机械可查）**：断言若读某 h-in-λ 通道的样本读数
（命中次数/单次采样值）而非其 h-不变式投影，为待纠形态。

**（2026-07-15 用户定稿纠正）**：两种 h-不变式形态**均可构造**、无需新的结构化断言通道（单元 D）——
成员归属＝`abs_found(每成员 IP)` 合取（`⊆` 由 config 闭集保证、不需断言）；分布＝大样本发包＋
`show statistics` 读每后端累计命中＋区间正则（`domain_grammar.json` 分布类 provenance 已载此法：
「发 N 次→各后端累计命中∈统计区间，守恒 Σ==N」）。故实现**不是 emit 期 reject-and-strand**（那会把
分布类案从假 pass 变永久卡死），而是**把既有方法摆给 worker 构造侧**（现仅喂了 grade_extract 检测器、
未喂 worker）；「样本不足 vs 缺陷」的分界判据（本节上文「欠定/缺陷分界」句：h 边缘化后统计量仍偏离=缺陷候选）落在 **attributor 自查**（摆事实＋prompt
自问，非机械翻转门）。写回护栏＝**像记忆一样如实标血统＋用前先核**（(45) 的实现，非「不写」）。

## 2. 四算子代数与 clear 作用域格（公式 (29)）

**(29) 四算子及其律**：
```
config X   : 状态变换 s → s'
no X       : 配置面对象级复位候选 —— 作用域 ≡ X(手册签名配对,恒不扩大);
             no X ∘ X = reset_X(回对象默认态;非群逆:复位后子状态可与原态

clear X    : 子空间幂等复位 —— clear X ∘ clear X = clear X
```

> **复位候选语义边界注（2026-07-17 对齐 K 头部 07-13 形式化加固 + Py-Eng 实现交叉验证）**：
> 三个成分逐一有据——①**候选而非定理**：`no X` 是复位**候选**，是否真使 diff 清零由执行后
> **复探裁决**（K §2.7.7 R4-G3 同义：机械回放后必跑验证探针，失败→合法 ask——理论与实现
> 在此精确同构，`bed.py` 复探链/`tau_coverage.py` 消费均按保守语义，不预设复位成功）；
> ②**作用域恒等**：作用域 ≡ 原命令 X（`inverse_forms` 从 command_inventory 签名配对现查，
> 闭合于手册版本，恒不扩大——destructive 升级路径在文法层即不可表达）；③**非群逆**：恢复的是
> 「对象默认态」非「作用前原态」，对象在 X 前已带非默认子状态时 `no X ∘ X ≠ id`（群逆恒等式
> 对 CLI 过强，原式已废）。τ 责任集与 `inverse_forms` 一律按此消费（差集内**己方创建**的写→

**clear 作用域格**（包含链）：`clear <feature>`（单功能域：slb 92/statistics 76/
sdns 71/http 40…）⊂ `clear config primary/secondary/file`（配置层）⊂
`clear config all` ⊂ `clear config factorydefault`（出厂）。

推论：L7 fail 有跨层候选根因集=其祖先链——归因难度的数学本质。

## 4. 传播定理与复位差集定理（公式 (31)(32)）

**(31) 传播定理**：写 Lk 层对象 o 的影响集 = o 在实例图上的**上闭包**（所有经 o
承载的高层对象）；**扇出反比于层**（一个 L2 接口承载 N 个 L3 地址×每地址 M 个
L4/L7 服务）——越底层的写，爆炸半径越大。
实证：233 改 L2（vlan/port2）→ 34.70@L3 断 → 9 个 L7 案全灭——一次 L2 写的上闭包
覆盖了本域最大的一簇 fail（233 家族，具体占比的复现依赖已丢的分类脚本，此处只作定性）。

**(32) 复位差集定理**：污染逃逸面 = 用例可写空间 W − per-case 复位空间 R。框架
per-case 清理=clear slb/sdns（L4/L7 功能域），L2/L3 对象（interface/vlan/route）
属系统配置章、不在任何功能 clear 作用域——**差集恰好集中在扇出最大的底层**。
「共享床上底层写最危险」不是经验而是 (31)×(32) 的交：逃逸面与爆炸半径在低层重合。

**复位差集是可消除的实现缺陷、非自然律**（2026-07-13；接 §0.4 τ 派生 + (29) 逆元 +
DESIGN §18.4/S4）：W−R 中 **R 是设计可扩张量**——per-case 复位不限于框架功能域 clear。
**配置面差集可经 (29) 案级 no 复位算子机械回放清零**（σ 的配置写→逆推 no 序列，形态从手册
签名派生；复位语义对己方创建的差集内写=清零成立，见 (29) 边界注；DESIGN §18.4 `inverse_forms`
数据已落、S4 恢复优先序已定「机械逆放先行」）。
推论：**用 clear／LLM 自造恢复去擦配置面污染是可避免的犯规**（且 clear 有向 `clear config
all` 升级的毁灭路径，§18.4.1）——非「设备脏、oracle 难」，正解是引擎确定性派生 no 逆元、
worker 无从选 clear。**诚实边界**：逆元只清配置面（R6-G6）；状态面残余（会话表／统计／
持久文件／对端，及他人写）逆元够不着，仍归 infra 床账／快照——§15「不动清单」的床账因此
**保留、非退役**。故 τ 案级负担与床账是**随 R 扩张的分面降级**（配置面随逆元退场、状态面
残余处保留），非整体删除。工程目标（**未验收**）：配置面差集 measured→0 且关该机件无 T1
回归（接 §18.4／S4 兑现② 与 #76 残余）。

## 5. 观测结构与归因距离

- **投影核**：6 个顶层对象无 show 投影+跨设备盲区（触发端的 L2 邻接形态——tagged/
  untagged——不在被测设备的任何投影里，233 首轮实证）——不可观测分量的污染无法用
  快照 diff 捕获（infra 床态快照的能力上限，此处给出其数学根源）。
- **归因距离**：断言几乎都打在 L7 回显，根因可在 L2-L7 任一层；归因成本 ∼
  断言层与根因层的**链上距离**。推论：正确的归因结构=**沿承载链下降**（L7 断言
  fail → L4 在听吗 → L3 可达吗 → L2 邻接对吗），每层用该层 show 投影，是机械探针
  序列——attributor 现有「版本层下降」之外缺的正是 OSI 链下降。
- **π 的忠实实现（2026-07-15 范式对标补，见 K 文档 §2.13 / DESIGN §18.15）**：投影核 π 的
  工程实现是一个**解析器**——把 show 回显解析成结构化对象、断言打在对象的 **key-path** 上。
  raw-text 子串匹配（框架 `found/not_found`，`re.DOTALL` 无 MULTILINE、窗口=命令回显+数据+
  提示符整块）是 π 的**不忠实近似**：断言绑的不是 π(对象)、而是含命令回显的原始窗口——这是
  window 失真（DESIGN §18.10）与恒真断言（断言命中命令原文，§18.14 六门）两族崩溃的**共同
  数学根源**。忠实 π 需两条：①结构化解析到对象、断言绑 key-path（外部现成：Genie
  `device.parse()`→dict + Dq `value_operator`、NAPALM getter+key-path+算子）；②**fail-loud**
  ——解析器遇不认识的回显必须显式报错、不得静默错位（Genie 血教训：Loopback30 的 IP 被误挂到
  Loopback10——**错位的对象比 raw 文本更难抓**，因为它看似合法结构）。∴ π 解析器 schema 必须
  锚手册/footprint、禁 LLM 现编 key-path。OSI 链下降每层的 show 投影同理走结构化 getter。

## 10. 时间轴：候选公式 (34)（2026-07-12 缓立——数据不支持即停的示范案例）

**候选命题**：状态空间分**配置面**（本案算子写之间静止，断言时不变）与**状态面**
（自治演化：TTL 过期/health check 定时器/会话老化/统计累积/RR 轮转态）；

```
(34·候选) 对状态面的断言是时间索引断言 O ⊨_t E ——
          合法形态 = 等待-验证对 ∨ 容差窗,窗宽从该对象的时间常数导出
          (时间常数在手册签名的 interval/timeout/ttl/expire 参数里,类型级可解析)
```

**为什么只是候选（检验一，`RESEARCH_theory_data_checks.md §1`）**：yzg 数据不
支持——①sleep 的时变邻近富集是块大小混杂+单签名饱和假象（分层后归零），且卷面
sleep 全是「等配置生效」型；②翻转实例中有直接落在配置面断言的反例，翻转主导解释
是床污染（infra T1），无一例归因自治演化。唯一强定性支持（zhaiyq persistence TTL
时序翻转）原始卷不可得。（具体计数依赖已丢脚本、不可复算，此处只保留定性结论。）

**域外背书充分**（文献调研二）：safety/liveness 二分（TLA+）、flaky 第一根因
async wait 45%（FSE'14）、QuickLTL（PLDI'22，含警告：无界 eventually 在有限
trace 退化恒真——与 not_found 恒真同族）、RFC 6412 收敛时间标准化、Awaitility。
即：**理论方向有工业/学术三层背书，但「它是本域主要失效模式」的域内证据未至**
——yzg（sdns listener 配置域）天然偏配置面，没给该命题施展对象。

**处置**：①不立门不进 emit；②手册时变签名表（带时间参数命令，以 §0.3 可复现的
3572 签名解析为基现筛——旧「202 条」计数不可复算已废，见 §9）作为文法层数据随类型图
解析（DESIGN §15 S1）捎带落盘——零代码增长，数据先备好；③转正前置条件=一个
persistence/health-check 域批次的原始数据支持预言②；④sleep 治理维持队列末位。

**多写者边界（S8，检验一附带修正，即刻生效）**：本文档「配置面在算子写之间
静止」的「算子」限定为**本案算子**——共享床是多写者，他案写造成的配置面变化
归 infra T1 域（床账/hermetic 辖），既不是 (34) 的反例，也不在其辖区。单案
建模视角的这条边界此前未声明，是检验一 2 例「配置面翻转反例」的正确归档位。
