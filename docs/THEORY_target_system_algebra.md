### 0.1 原始对象

设备是不可直视的机器，能做的只有发输入、收输出：

```
M = (S, U, Y, δ, λ)   Mealy 机
S 状态(不可直读)  U 输入(CLI 命令与网络包同类——都是输入)
Y 输出(回显与响应包)  δ:S×U→S 转移  λ:S×U→Y 输出
```

### 0.2 算子分类的相对性（四算子降为定理）

config/no/show/clear 不是原始概念，是 U 按 (δ,λ) 性质的等价类，且**分类必须
相对于状态分解定义**：dig 对配置分量是纯读（δ 恒等），对运行分量（统计/会话表）
是写。「激励 vs 观测分不清」的病根即在此——分类不是输入的内在属性，是输入
相对于状态子空间的属性。故必须先推导状态分解。

### 0.3 状态空间的三轴分解（面/层/域首次获得数学定义）

分解的唯一合法依据是**动力学性质**（手册章节是文档组织，是派生物非依据）。
三轴是三个**独立定义**的划分（每个分量同时携三标签；「正交」指定义相互独立，
非统计独立——相关性本身是内容，见轴三）：

- **轴一（稳定子分解）→「面」**：按「谁使它变化」——x_cfg 只被操作者输入改变
  （配置面）/x_run 被流量与时间自发改变（运行面）/x_persist 被显式持久化操作
  改变、跨重启存活（持久面）。G6 的二分是其粗化。
- **轴二（影响 DAG 分级）→「层」**：分量间影响关系生成 DAG；**OSI 层不是公理，
  是「该 DAG 近似分级」的可检验经验命题**。(31) 上闭包=DAG 可达性。
  **预言2 已坐实**：3572 签名（解析覆盖 99.64%）建边，配置命令跨层边低→高
  98.4%（373/379），6 条反向边可枚举且全为 IP 字面值参数（剔除该口径后 100%）
  ——反向边白名单：低层健康/过滤行为引用 L3 字面值（bond health / bridge
  apprule 族）。
- **轴三（交换中心分解）→「域」**：子空间独立 ⟺ 各自输入可交换（交换子为零）；
  **功能域=极大可交换分解的块**。硬推论：**跨域干扰 ⟺ 两块共享底层分量**
  （交换子非零）。**预言1 已坐实（⇒ 方向）**：yzg 三轮 51 个互扰实例 100% 被
  「写实体 ∩ 触碰实体于底层/持久分量」命中（承载图闭包版——655173 的 IPv6
  受害与 vlan100 污染不同名但同承载 port2，闭包命中词法漏）；run10 净床 0 受害
  =干净对照。**⇐ 方向降为必要条件**（精确率 74-80%，假阳主型=共享但观测不
  依赖该分量数据面）——充分化需补「受害方观测 π 依赖该分量所在面」细化条件，
  升定理需多污染源实验（预注册）。
- **域的纤维性**（考古提炼保留）：脑图一级节点=手册功能域章=轴三块=footprint
  前缀组织——四层表示共享同一分区单元；意图低秩（r_eff≈活跃域数）、写回增益
  按域、跨脑图零重合/域内骨架重合 45-51% 皆由此结构必然。

### 0.4 测试用例=序列成员的完整形式（六元组，公式 (39)）

S 不可直读，只有观测等价可谈。检验「场景 X 下行为为 Y」，从可辨识性推导出
五个元；用例还是**实验序列的一员**，序列成员多一条构成性义务——把现场还给
下一个实验。完整形式：

```
(39) case = ⟨s₀, σ, e, π, R, τ⟩
  s₀ 初始条件(必须已知,否则观测无法归因)
  σ 制备(把状态送进 X 的观测等价类)   e 探测输入
  π 输出采集(受投影核限制)           R 预期约束
  τ 还原(恢复算子序列;数学地位:s₀ 是我自己的前置,τ 是下一个用例的 s₀ 的前置)
     序列可组合性条件: τ∘exec ≈ id (在共享分量上)
```

**τ 的分工与派生**（卫生段权威分离定理的执行面）：框架 per-case 清理=τ 的
功能层子集（clear slb/sdns）；**案级 τ 负责复位差集部分**（(32)：L2/L3 写、
持久面写——正是框架够不着的分量）。τ 是实验室规程非实验设计——权威在引擎、
**机械可派生**：从 σ 的差集内写逆推 no 序列（(29) 逆元的案级应用），不需要
作者在脑图里写。工程推论=**配对恢复门**（emit 期：卷面含差集内写而无对应
恢复步 → 呈报，形态同 S6 不硬拒）。

**τ 缺元的教训（C14 案例）**：2026-07-12 上午的初版推导只有五元组——从
可辨识性单视角推，漏掉了序列成员视角；τ=setUp/tearDown 是 xUnit 七十年代
定型的工业常识，R8 八攻无一命中此缺元（自我对抗盲区共享），由 run12 实弹
（655203/233 自污染五次循环）+用户点破+范式对标暴露。**实证**：run12 全部
三个真缺陷（retry 被闸吞/面板缺重编出口/污染者可交付）的共同上游=τ 不在
本体 → L 无 teardown 槽位 → worker 结构性编不出恢复 → 门无物可检。

**s₀ 是一等成员**——此前一切提炼漏掉它，而它恰是 T1 占 54% 的位置：污染的
数学定义=**s₀ 假设违背**；复位差集=未被任何复位算子覆盖的 s₀ 分量；hermetic/
床净不是外来最佳实践，是**观测归因的前提条件（推导所得）**。**预言3 已坐实**：
按「制备链触及复位差集」机械标记 s₀ 危险案，历史污染者查全 4/4=100% 且版本
分辨正确（655233 仅 run7 污染版被标——规则跟卷面走非案号黑名单）；查准 57%，
3 误报全为「触及但被自清理/框架扫尾/未到发作条件挡住」——与「上界」语义一致。
**新结构认识（保存族反扫）**：持久面快照（write memory/file）把功能域残留冻结、
config 恢复时复活——**跨面洗白路径**，绕过 per-case clear 的功能域作用域；
「全局配置存储」是持久面的一个分量，写它=触碰全机状态。
**卫生段权威分离定理**（对应表提炼保留）：脑图承载实验设计（权威在作者），
复位/卫生承载实验室规程（**权威在引擎**）——引擎机械补全清理不构成改写作者
意图，不触 ask 边界（S4 逆元回放的合法性从用户裁决升为定理推论）。

### 0.5 隐藏自由度 h：欠定与 flaky 的统一

真实输出 y=λ(s,u,**h**)，h=既不能制备也不能观测的自由度（RR 轮转指针/调度/
时钟/并发/共享床上他人的写）。分类按 **h 的位置**：

```
h 在 λ 内   → 欠定:R 真值本质依赖 h → 对策=R 边缘化为 h 不变式(区间/集合)
h 在 s₀     → 床污染:再分【己方写账(可机械逆放)|他人写(共享床治理)】(B6 子轴)
h 在 π 通道 → 取证/传输噪声 → 复跑/通道修复
无 h        → 确定性错误 → 出本框架辖区,归 Ω 判定树 + is/ought 仲裁
```

**机理推论**：「复跑可救」是 h 位置的推论——π 噪声复跑=h 重采样可救；s₀ 污染
复跑 h 冻结在脏床不可救（除非清场）。**欠定/缺陷分界**：单次观测依赖 h=欠定；
h 边缘化后统计量仍偏离=缺陷候选（wrr 配比异常的正确归档）。**预言4 已坐实**：
8 个历史误归案重标全面占优——env_blocked 表达不了「批内前驱污染」、668030
同一签名三 run 三标签（按 π 对策治 s₀ 病）、同症异判在 h 轴结构性不可能。
**可判性三档（诚实）**：h-in-λ 与无 h 离线即判；s₀/π/真环境故障**单案观测等价**
（B1），判定成本 O(批)——批级算子（同批 pass 对账/床账/跨轮对照）大半已在，
这正是「单案填词表必然误归」的结构原因。边界 B1-B7 详见
`RESEARCH_theory_data_checks.md §5.5.3`。

### 0.6 K=模型，编译=规划（中心论断的推导版）

K 是对 M 的部分模型（U 合法子集/δ 效果/λ 形态/影响 DAG），来源只有文档（对 M
的声明，可错）与实验（对 M 的采样，带 h 噪声与 build 时效——stale 的推导定义
=M(build) 时变而模型未更新）。编译=在模型上做规划（为 D 找 ⟨s₀,σ,e,π,R⟩）。
**规划质量 ≤ 模型质量是规划的定义性质**——K 理论中心论断 (9)「prompt 为低阶项」
由经验命题变为推论；LLM 是规划器，换更强规划器突破不了模型上界。

## 2. 四算子代数与 clear 作用域格（公式 (29)）

**手册形态学统计**（10.5 CLI 29 分册）：no 命令 633 条、show 命令 899 条、clear
命令 605 条；61 个顶层命令名中 55 个有对应 show 投影。

**(29) 四算子及其律**：
```
config X   : 状态变换 s → s'
no X       : 配置面逆元 —— no X ∘ X = id  (限配置面,见 R6-G6 边界)
show X     : 只读观测投影 π_X(不改 S;61 顶层中 55 个有投影,6 个在投影核外)
clear X    : 子空间幂等复位 —— clear X ∘ clear X = clear X
```
**clear 作用域格**（包含链）：`clear <feature>`（单功能域：slb 92/statistics 76/
sdns 71/http 40…）⊂ `clear config primary/secondary/file`（配置层）⊂
`clear config all` ⊂ `clear config factorydefault`（出厂）。

**逆元律的诚实边界（R6-G6）**：no X 恢复的是**配置面**，不恢复**状态面**（会话表/
统计计数/磁盘持久文件/对端已同步内容）——「持久化家族」的数学定义即**状态面写**
（配置面逆元对其无效的算子），不再需要枚举通道：save/synconfig/segment 文件全部
落此定义。状态面恢复归 infra 域（快照/T4/床账）。

## 3. 承载合取律（公式 (30)）

数据面包流：触发端 →(L2 邻接/vlan)→(L3 可达/ARP)→(L4 监听)→(L5-7 服务逻辑)→ 返回。

**(30) 承载合取律**：Avail(o@L7) = conf(o) ∧ listen(anc₄) ∧ reach(anc₃) ∧
adjacency(anc₂)——功能可用性是其在实例图上全部祖先的合取。**一个 L7 断言
（check_point）是对整条承载链的合取断言**，与三合取定理（§2 横向）同构，方向为纵。
推论：L7 fail 有跨层候选根因集=其祖先链——归因难度的数学本质。

**冗余修正（R6-G2）**：HA 双机、多 listener、lastresort 等冗余结构下，合取在冗余组
处变析取（k-of-n）——精确形态是 and/or 图。第一版取合取近似（过报不漏报），呈报时
声明近似；冗余组建模待实例图成熟后补。

## 4. 传播定理与复位差集定理（公式 (31)(32)）

**(31) 传播定理**：写 Lk 层对象 o 的影响集 = o 在实例图上的**上闭包**（所有经 o
承载的高层对象）；**扇出反比于层**（一个 L2 接口承载 N 个 L3 地址×每地址 M 个
L4/L7 服务）——越底层的写，爆炸半径越大。
实证：233 改 L2（vlan/port2）→ 34.70@L3 断 → 9 个 L7 案全灭，40/73=54% 的 fail
是一次 L2 写的上闭包（R5 定量）。

**(32) 复位差集定理**：污染逃逸面 = 用例可写空间 W − per-case 复位空间 R。框架
per-case 清理=clear slb/sdns（L4/L7 功能域），L2/L3 对象（interface/vlan/route）
属系统配置章、不在任何功能 clear 作用域——**差集恰好集中在扇出最大的底层**。
「共享床上底层写最危险」不是经验而是 (31)×(32) 的交：逃逸面与爆炸半径在低层重合。

## 5. 观测结构与归因距离

- **投影核**：6 个顶层对象无 show 投影+跨设备盲区（触发端的 L2 邻接形态——tagged/
  untagged——不在被测设备的任何投影里，233 首轮实证）——不可观测分量的污染无法用
  快照 diff 捕获（infra 床态快照的能力上限，此处给出其数学根源）。
- **归因距离**：断言几乎都打在 L7 回显，根因可在 L2-L7 任一层；归因成本 ∼
  断言层与根因层的**链上距离**。推论：正确的归因结构=**沿承载链下降**（L7 断言
  fail → L4 在听吗 → L3 可达吗 → L2 邻接对吗），每层用该层 show 投影，是机械探针
  序列——attributor 现有「版本层下降」之外缺的正是 OSI 链下降。

## 10. 时间轴：候选公式 (34)（2026-07-12 缓立——数据不支持即停的示范案例）

**候选命题**：状态空间分**配置面**（本案算子写之间静止，断言时不变）与**状态面**
（自治演化：TTL 过期/health check 定时器/会话老化/统计累积/RR 轮转态）；

```
(34·候选) 对状态面的断言是时间索引断言 O ⊨_t E ——
          合法形态 = 等待-验证对 ∨ 容差窗,窗宽从该对象的时间常数导出
          (时间常数在手册签名的 interval/timeout/ttl/expire 参数里,类型级可解析)
```

**为什么只是候选（检验一，`RESEARCH_theory_data_checks.md §1`）**：yzg 数据不
支持——①sleep 的 92% 时变邻近富集是块大小混杂+单签名饱和假象（分层后归零），
且 13 个 sleep 全是「等配置生效」型；②8 个翻转实例中 2 例翻在配置面断言（直接
反例），翻转主导解释是床污染（infra T1），0 例归因自治演化。唯一强定性支持
（zhaiyq 22/50 persistence TTL 时序翻转）原始卷不可得。

**域外背书充分**（文献调研二）：safety/liveness 二分（TLA+）、flaky 第一根因
async wait 45%（FSE'14）、QuickLTL（PLDI'22，含警告：无界 eventually 在有限
trace 退化恒真——与 not_found 恒真同族）、RFC 6412 收敛时间标准化、Awaitility。
即：**理论方向有工业/学术三层背书，但「它是本域主要失效模式」的域内证据未至**
——yzg（sdns listener 配置域）天然偏配置面，没给该命题施展对象。

**处置**：①不立门不进 emit；②手册时变签名表（202 条）作为文法层数据随类型图
解析（DESIGN §15 S1）捎带落盘——零代码增长，数据先备好；③转正前置条件=一个
persistence/health-check 域批次的原始数据支持预言②；④sleep 治理维持队列末位。

**多写者边界（S8，检验一附带修正，即刻生效）**：本文档「配置面在算子写之间
静止」的「算子」限定为**本案算子**——共享床是多写者，他案写造成的配置面变化
归 infra T1 域（床账/hermetic 辖），既不是 (34) 的反例，也不在其辖区。单案
建模视角的这条边界此前未声明，是检验一 2 例「配置面翻转反例」的正确归档位。

## 11. 写者全集公理（公式 (38)；2026-07-12 R10 审计——词表封闭性盲区收口）

```
(38) Writers(S) = {案级 CLI 写(σ/τ), 流量写(e 的 δ 副作用:缓存/会话/统计),

```

- **各写者的账务地位**：案级写→τ 义务+床账；框架写→契约测试钉行为（infra §6）；

  缓存/会话残留互扰实证）。
