## 根因(定位到行)

`_s0_pair`(`nodes.py`)的 L2/L3 分支:`shared = p_ents & vict["entities"]`——两案
entities 有交集即判前案为 polluter。但 entities 里含**测试床固定基础设施 IP**:
- `172.16.35.231` = dig 的固定后端服务 IP(几乎每个 sdns 案的答案);
- `172.16.32.70`/`172.16.34.70` = 4 个白名单 port 接口 IP(基线面,框架不清、每案共用)。

667986 自己的满配 listener 就配在 172.16.32.70 上;655203 也用 172.16.32.70(bond)。
共享的是"大家都用的固定 IP",不是"655203 写进去污染 667986"。∴ 假阳。

## 真 fail(被假阳掩盖)

667986 断言 `(sdns listener \d+\.\d+\.\d+\.\d+ \d+\n){16}` 要 16 条**连续带端口**;
但输出 `sdns listener 172.16.34.70`(无端口,默认53)与 `…70 10001`(带端口)混排,
无端口条打断连续匹配 → 凑不满 16 → fail。**这是 V 层断言写法问题,该走深归因修
断言(如 `(sdns listener \S+.*\n){16}` 或按实际形态),却被 s₀ 假阳拦去 bed 呈报。**

## 修法(类比 persist 收窄,机械闭集)

L2/L3 共享实体判据排除**测试床固定基础设施 IP**——从 `network_topology.json` 的
固定地址集(接口 IP/port IP/后端服务 IP/网关/白名单)机械构造排除集,`shared` 减去
它:只有**非基础设施的、真正被前案新写入的**共享实体才算污染候选。数据驱动零硬编码
(新增基础设施 IP = 改 topology JSON)。同 persist 的"框架清不掉 ∧ 真污染路径"精神:
固定基础设施 IP 是所有案的合法共用对象,不构成"前写后读脏"。

## 影响面(诚实)

- **不是 §18.13/§18.12 的回归**:persist 分支没判它(655203 persist 空),是既有
  L2/L3 分支;§18.13 三元组题面主验收不受影响。
- **代价**:真问题(断言缺陷)被掩盖 + 多问一个假 bed 题(用户答非污染/复跑放行,
  但 667986 的断言仍需修——会在下一轮以同签名再 fail,届时若仍判 s₀ 则冻结)。
- echo=necessity_only + bed 呈报是引擎诚实兜底(必要非充分,不静默交付),所以不
  投毒,只是问了不该问的题、掩盖了该修的断言。
