## 四、贯穿两批的失败模式（供后续收紧参考）

1. **分布 / 算法类构造是最大难点**（dongkl 6/12）。分类这轮判对了（采样 vs 成员），但「发多少包、断言什么形态、命中区间还是精确、时序锚点能否成立」的**构造**仍常错。→ 未来 `compile-worker.md` 收紧方向（哪条命令显什么、区间 vs 精确、前置序、时序锚点与算法匹配性）。
2. **设备拒命令多是前置/参数缺失**：host 未建就绑 pool、wrr 缺 priority。属编写孔可自查、可修的构造错。
3. **环境 / 床结论翻转**（778041、994957）：复跑不稳、执行失败标记致断言落空——需床稳定性 + vacuous 检测（原理层 (44)）继续兜。
4. **已知缺陷 / 环境资源缺口**（105941 cname 池缺陷 149877；yzg 655233 VLAN 需实机 IP + trunk）：非编译能解，出口是缺陷候选单 / 环境补齐，引擎正确地挂起而非硬跑。
