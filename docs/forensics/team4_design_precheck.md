### 1.1 两个结构的消费者全集（子agent全景，亲核锚点）

**`ask_user_answers.jsonl`**（写题干全文+answers，append-only）：

**`folded_into`**：

### 1.2 头号红线（任何 fold 候选必须守）——隐性跨台账不变量

先问后落校验 = **按成员案逐 member 过门**（`_land`→`compile_user_decision(autoid=member)`→门），但过门证据只有**代表案一行 ask_user_answers**。**桥接不变量 = "代表题面嵌入全体成员尾6位"**（nodes.py:649/653-655 手工延续 `tails=各成员[-6:]` 拼进 q["question"]）→ 落台账那一行 → 门子串匹配每个 member 尾号。

**破坏此不变量 = 逐成员门假拒 → `_land` 返 False → 案回 needs_decision → 下轮重问 → 活锁**（正是 nodes.py:644/verifiability_tool.py:445-450 反复告警的失败类）。**任何改 fold 题面组稿 / 扇出 keying / 改逐成员问 的候选，必须保住此不变量**——这是 folding×先问后落门评审的核心判据。

### 1.3 两个设计脆弱点（既有，fold 放大）

1. **先问后落门用子串匹配而非结构化 JSON**（verifiability_tool.py:415 `aid[-6:] in _line`）——尾6位（如 655233）可能撞题面里别的数字子串（同型 `[[transient-error-bare-digit-marker]]` 裸数字 marker 误中 autoid 教训）。fold 一行含多个尾号，**撞车概率随组大小上升**。候选若增大折叠组，此脆弱点放大——建议评审要求门改结构化匹配（题面独立字段带 member 列表，非子串扫）。
2. **adopt 免问采用也经 _land→门**（nodes.py:619）——adopted 案本轮未被问，其过门依赖 ask_user_answers **持久台账**里恰有**历史**尾号命中。若持久台账被清/轮转，adopt 过门可能假拒。**单列复核项**：adopt 路径的先问后落门凭什么放行（历史台账命中 vs 特权豁免）。
