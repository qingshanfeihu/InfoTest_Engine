# V8 交互路径 UX 验收基准（User observer 挑刺基准 · Design 三维验收点）

> 对应 Theory `team4_interaction_path_completeness.md` 22 路径（P1 bed_gate×2 / P2 ask_decision×17 / P3 ask_contradiction×3）。
> 用途：**User observer 冒烟时按此挑刺，不靠临场感觉**——每条路径实例化「该看懂什么 / 该会答什么 / 什么算看不懂」，命中「看不懂」判据即报 leader。
> Design 出（2026-07-18，UX 维度）；zhaiyq 起批前交付。★=zhaiyq 大概率出现、优先。

## 三维验收点定义（通用判据）

| 维 | 验收点（面板该达到） | 「看不懂/不会答」判据（User observer flag） |
|---|---|---|
| **该看懂什么** | 题面传达三件：①这个用例**什么问题**②**为什么问我**（不是引擎自己能定的）③我的选择**影响什么** | 读完题面 get 不到「问的是什么/为什么找我」= 看不懂 |
| **该会答什么** | 选项让用户**据用例本身可判**（无需 V8 引擎知识）+ 每选项带「**对你的用例**意味什么」后果（D2） | 选项要懂引擎机制才能选 / 后果不明 / 选项是内部动作名 = 不会答 |
| **什么算「看不懂」** | 题面/选项**无**：引擎黑话（术语/机读码 nd:/autoid/facts.jsonl/claim_kind）、内部动作名（未解释的"改过程"）、截断半句、英文枚举 | 命中任一即 flag（与 leak_scan/D2/D4 同源判据） |

## P1 bed_gate（床态体检，nodes:298）

### ★P1-a/b 床态体检 → 继续(proceed) / 停止(stop)
- **该看懂**：这个测试床现在能不能跑这批用例——**体检发现了什么**（残留配置/进程/网络不可达），**为什么要我拍板**（引擎不敢擅自清非己方残留，INV-9 床权）。
- **该会答**：`继续`（床没问题/残留可忽略，照跑）/ `停止`（床有问题，我去修床、改参数同批重跑）——用户据「床态描述」判，不需懂床治理机制。
- **看不懂 flag**：床态用机读（裸 IP/子网/`pytest.*ist_staging` 进程名）不转人话；「停止」后果不说（停了怎么继续）；残留归属不清（谁的残留、为什么问我）。

## P2 ask_decision（nodes:774，承载最多问题种类）

### ★P2-a/b/c 欠定三选（改过程 / 改预期 / 改描述）
- **该看懂**：这个用例**哪里没定清楚**——①验证方法缺（没说怎么验证命中）②预期值不可验（断言的值这个床测不出）③描述本身模糊；**为什么问我**（引擎按现有描述编不出真覆盖行为的用例）。
- **该会答**：`改过程`=让引擎补验证步骤（引擎按你的意图加观测/断言）/ `改预期`=改断言的值（你给能验的预期）/ `改描述`=你回去改脑图重来。**每选项「对你的用例」**：改过程→引擎补步骤本轮重编；改预期→按新断言重编；改描述→本轮挂起、你改脑图后下批继续。
- **看不懂 flag**：欠定点用术语（`h_absent`/`claim_kind`/`verification_path_absent` 机读）不解释「哪里没定清楚」；「改过程/改预期/改描述」只给动作名不给「对你的用例」后果；题面含 nd:/autoid 机读锚。

### ★P2-d/e/f test_point 采纳（采纳该等价方案 / 我给别的 / 挂起）
- **该看懂**：这个用例**原来的验证方法这个床跑不了**，引擎**建议了一个等价方法**（能在这床上验同样的行为）；**为什么问我**（换验证方法要你确认等价、不擅自改）。
- **该会答**：`采纳该等价方案(方法见题面)`=用引擎给的等价方法重编 / `我给别的等价方案`=你自己填一个 / `挂起`=先不弄。**「对你的用例」**：采纳→引擎按等价法编、和原方法差异在交付报告声明；自给→按你的填法编。
- **看不懂 flag**：等价方案含设备命令黑话（`show sdns`/`dig` 语法）不解释「验的是什么」；label 截断半句（旧 `采纳「…作为sdns liste`——F-TUI-2 已修成固定短语，回归即 flag）；「方法见题面」但题面方法也是黑话。

### ★P2-l/m/n/o 挂起处理（suspend / keep / resume / defect）
- **该看懂**：这个案暂时定不了/你想先放放——**当前是什么状态**（挂起/恢复/确认缺陷）；**resume(P2-n)** 特别：你上次对此案裁过什么（题面带 prev_dec 上下文）、现在**接着修**。
- **该会答**：`挂起`=放到下批 / `保持挂起`=继续放着（不被恢复覆盖）/ `恢复处理`=接着修（引擎重新问你欠定）/ `确认缺陷`=这是产品 bug、记入候选单。**「对你的用例」**：挂起→下批再问；恢复→本轮重新走欠定问你；确认缺陷→不再重编、进缺陷候选单交人核。
- **看不懂 flag**：resume 恢复后**没反应**（effective=false「恢复了个寂寞」——#37 已修，回归即 flag）；「保持挂起 vs 挂起」区别不清；确认缺陷后去向不明。

### ★P2-p 折叠广播（gather，成员>1）
- **该看懂**：**这几个用例是同一个问题**，一次问、你的答案**广播给全组**；**为什么一起问**（同诊断依据+同污染者集，逐个问是重复）。题面**列出成员**（尾号，非 18 位）。
- **该会答**：答一次、扇出全组——用户知道**这个答案影响哪几个案**（题面成员清单）。
- **看不懂 flag**：成员用 18 位 autoid 不转尾号（认不出是自己哪条）；广播范围不清（答了影响几个）；折叠依据不说（为什么这几个算一组）。

### P2-g~k / P2-q（cap / env / bed / Other，简表）
| 路径 | 该看懂 | 该会答（+后果） | 看不懂 flag |

## P3 ask_contradiction（nodes:2396，矛盾即问）

### ★P3-a/b/c contra 矛盾（重排复验 / 如实降级 / 确认）
- **该看懂**：这个用例**单独跑能过、整卷一起跑却挂了**（矛盾）——**为什么不一致**（前面某个案改了床状态影响它）、**为什么问我**（引擎给了假设但要你定怎么办）。题面带**案件时间线+引文对照**（哪个前驱案影响的）。
- **该会答**：`重排复验`=换用例顺序再跑一次看稳不稳 / `如实降级`=承认这案不稳定、记入未通过卷（不假装过）/ `确认(按引擎理解)`=引擎的假设对、按它写判例。**「对你的用例」**：重排→再跑验证；降级→如实报告不入交付卷；确认→写回判例下批免问。
- **看不懂 flag**：矛盾用术语（`contradiction`/`reconcile`/复验机读）不解释「单跑过整卷挂」；前驱案用 autoid 不转短号；选项后果不明（降级=这案就废了吗）；判例写回术语。

## 附：判定式渲染 vs LLM 转述（D17 关联，用户面数据纪律）

**用户面的数字/计数/分类**（"N 个用例、X 通过、Y 挂起"）必须走**引擎判定式渲染/照抄 engine_report 机械事实**——**禁 LLM 转述时自行归类计数**（D17 实证：LLM 嘴 4+4=8 vs 引擎判定式 25+1=26 对）。User observer 见**总结数字与 delivery_report/engine_report 对不上**即 flag（LLM 数错 vs 引擎判定式的对照是硬判据）。人话措辞可 LLM、数字计数照抄引擎——两者分离。
