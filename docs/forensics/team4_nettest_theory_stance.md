## ① L0-L5 自动化分级 → 外部背书注记，非新不变量

**定稿**：L0-L5（IETF draft-cui-nmrg-auto-test）是**外部成熟度语言**（汇报/专利叙事用），**不立新公式**（C14：无独立工程消费者的不占公式位）。

- **「特定域 L4」的形式化 = 既有构件合取的命名**，非新公理：
  ```
  特定域 L4 ⟺ (可导性定理 §3.2：该域内 fail 有限步可导到唯一根因)
            ∧ (ask 前置 §2.6.3：实然手段自主试尽，人只碰应然余项)
            ∧ (ask 定理 (19)：人介入 ⟺ 决策依赖应然 ∧ K_ought 检索余项非空)
            ∧ (中心论断 §4：K 在该域健康——L4 达成度是 K 域健康度的函数)
  ```
  L4 是既有理论的**涌现性汇报抽象**（emergent），不是新 axiom——「自主编译-验证-归因-重编闭环、人只在 is/ought 点介入」正是上述四构件合取的外部命名。
- **落点建议**：§2.13 成熟范式对标加一行「IETF L0-L5 ↔ 本项目自主构件」映射（同 §8-G7 YANG/§2.13 pyATS 背书体例）；准入报告与 #24 六裁决方案用 L 语言表述阶段（当前≈L3-L4，目标「特定域 L4」）。**零新不变量、零代码**。
- **与不变量体系关系**：L4 不进守门族（(16)/(48)/(18b)），它是**汇报层度量**不是**执行层门**——混入会误把汇报抽象当机械门（同 grade 之死教训：路由器/度量不当质量门）。

## ② 批级覆盖评估器 → 合取②的批级 breadth 扩展（满射性审计）

**定稿**：这是**真新构件**（我们现无批级覆盖），性质=**合取② `φ(D)⊆closure(E)` 的批级 breadth 满射性审计**。

- **性质**：
  ```
  批级覆盖 breadth ≝ 脑图测试点集 T_D --满射--> ∪_case closure(E_case)
                    每个测试点 t∈T_D，∃案 c 使 t ∈ φ⁻¹(closure(E_c))   （防漏编案）
  ```
  NeTestLLM 的 breadth/depth 两维里：**depth（每案断言真覆盖该点）我们已有**（记录类型 gap/单调门/验证盲区定理 §2，per-case）；**breadth（每个测试点都被某案覆盖）是新的**——现有 lint 只查单案内部，无「脑图有 N 点、卷面漏了第 k 点」的批级满射检查。
- **机械可判定边界（诚实红线）**：
  - **breadth 机械可判 ✅**：脑图测试点可结构化枚举（一级/二级节点）+ intent 盖章（§18.9）已把案链到 group_path → 满射性是集合覆盖机械核（测试点集 ∖ 已覆盖案集 = 漏编集）。
  - **depth 不可判 ❌**：「断言是否**真**覆盖该点」= 合取②全量 = **撞 C1 grade 之死**（§9-C1）→ 只用既有可判定投影（记录类型 gap/单调门），**不越 C1 红线**（不引 LLM 判 depth 充当质量门）。
- **落点**：立为新审计构件（消费者明确=防漏编盲区，非防错编）；理论归属 = 合取②批级投影 + 验证盲区定理 §2 的批级机械化。**与 (46)/INV 族**：非直接（(46) 管 ask 题面）；主锚是合取②+盲区定理，可挂一个 INV-breadth（满射残差清零才准交付，同 INV-2 残差门体例）。

## ③ NeTestLLM 错误分类学 → 我们是其超集，无盲区候选

**定稿**：NeTestLLM 三类 ⊆ 我们归因层+Ω，**无我们缺的类别（无盲区候选）**，反向我们更完备。

| NeTestLLM 类 | 我们对应 | 关系 |
|---|---|---|
| 语法(syntax) | G(^) 语法拒绝(compile_attribute 机械预判) | 直接对应 ✅ |
| 不支持命令(unsupported cmd) | command_existence 门(S §6 (33) 版本参数化，**emit 期前置预防**) | 对应且**我们更前置**（它 runtime 分类、我们编译期拦） |
| 配置(config) | P 错(Ω §3.1) + 状态层 show 对账(§3.2 判定树) | 对应 ✅ |

- **我们更完备的类**（NeTestLLM 无）：文件级崩溃（found_times 等编译缺陷）、**broken 第三态 (44)**（≙xUnit ERROR）、**infra 域 ⑥**（床污染/取证失真）、**欠定 ⑤**（h-in-λ）。∴ 归因层无需扩类。
- **一个 deliberate 差异（非盲区，是设计选择）**：NeTestLLM 把 config 预分类为独立桶（利于检索 config-fix）；我们把 config-语义归 **undetermined→LLM**、**不 pre-classify**——遵纪律 (5)「给 LLM 原始设备事实，别喂预消化关键字信号表」（pre-classify 有误分类风险，E_MARKERS 前科）。不 pre-name 是我们防误分类的**主动设计**，非盲区。
- **结论**：错误分类学对照 = **方向自信的外部印证**（架构对），归因层零改动。唯一可选借鉴：NeTestLLM「历史修复检索→提案」= 我们判例 adopt 的同类物，已有。

## ④ footprint 字段抽取正则 → 判例层新条目类型，封闭性成立，= S §5 π投影核渐进落地

**定稿**：footprint 沉淀「命令→字段抽取正则 R」= **判例层新条目类型**，封闭性与自愈四层**相容**，理论定位 = **S §5 π 投影核的判例层渐进落地**。

- **理论定位**：S §5（π 忠实实现）已裁：断言应打在**结构化对象的 key-path**、raw 子串是 π 的不忠实近似（§18.14 恒真断言族/窗口失真的共同根）。footprint 字段正则 = 对**高频命令**沉淀「raw→抽出字段」的微型 parser——是 π 结构化方向的**渐进落地**（先高频、判例层沉淀），非全面 Genie parser 切换（sdns/APV 无现成 parser 库、数千条不现实，survey 已诚实标）。
- **封闭性论证（判例层"唯一无限增长层"性质不破）**：新条目走既有判例层封闭环——
  ```
  入库：正则 R 对命令 C 的抽取正确性经 device_verified 实证（同既有 footprint 观察的验证门）
  检索：worker 编写期 footprint_lookup 查得 C 的字段正则
  消费：emit 断言打在抽出字段上（零代码，判例层数据驱动）
  ```
  与既有 footprint 观察**同型**（已验证产品事实的一种），不新增代码层、不破四层封闭。
- **层归属边界（防混层）**：R 是**设备输出格式的观察**（随 build 变）=**判例层**（经验性）；**不是** mirror 源码可推导物（那归**文法层**，闭合于版本）。字段正则跟设备走、非跟 mirror 走 → 判例层归属清晰，不与文法层混。
- **收益**：渐进消解 §18.14 恒真断言族的窗口脆弱面（断言从 raw 窗口移到抽出字段），零代码、数据层扩展。
