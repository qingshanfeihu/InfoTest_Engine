# 论文三层分解理论 ↔ 现有编译流程问题：完整对照

> 2026-06-16。把《意图实现映射：信息论分解》(G⊔E⊔V 三层) 的理论，对照本项目编译流程这几天实跑暴露的真实问题，逐条判断：**哪些问题论文理论能直接指导解决、哪些只能部分指导、哪些论文管不着（属工程/环境）**。
> 依据：论文成稿（`Downloads/files (2)/意图实现映射_信息论分解_成稿.md`）+ 实跑记录（`memory/compile-3txt-run-observation.md`、`docs/yzg_grade_vs_run_audit.md`）。不凭记忆，结论附定理/实证出处。

## 一、论文给了什么（可用的理论，一句话各）

| 理论点 | 内容 | 对实现的指导价值 |
|---|---|---|
| **三层分解**（§3.5 定义3.6/3.7） | 实现 token 按来源三分：G=骨架/文法（关键字、句法）、E=环境常量（IP/设备标识，查表可得）、V=业务语义（意图特有取值） | 给"哪部分怎么生成"一把分类尺子 |
| **主定理**（§3.7 定理3.10 + 推论3.11） | 总不确定性 = H_G + H_V'，E 段=0。G、E 段可由确定性机制（检索+查表）压到零熵；**只有 V 段必须 LLM 自由生成** | "什么该确定性产、什么该交 LLM"的定量边界 |
| **稳定性分解**（§3.7bis 命题3.25/推论3.26） | 自由采样跨次不一致率 = 三段碰撞熵之和。G/E 段交确定性机制后这两段不一致=0，只余 V 段本征方差 | 量化"全程 LLM 不稳定"的根源 = 在 G/E 段引入了本可消除的随机性 |
| **可达性上界**（§3.14 定理3.22） | G⊔E 段错误率 ≤ β_k（贝叶斯本征）+ (1-ρ_k)（前缀未覆盖）。**扩大先例库推高 ρ_k + 加深可信前缀压低 β_k 两个纯结构手段即可把 G/E 段错误驱至本征下限** | 先例库该多大、前缀该多深的定量依据 |
| **Fano 下界**（§3.10 定理3.15） | V 段错误率 ≥ (H_V'-1)/log M_V，实测 ≥4.2%。任何不访问意图语义的填充器必在 V 段出错 | V 段不可省的硬证明 |
| **可信窗口**（§3.8 + §4.9） | 单例率随前缀深度暴涨（depth-3 已 57%），深前缀低熵是过拟合。可信窗口 k≤2 | 先例检索/前缀匹配该用多深 |
| **离群度门控**（§4.10） | 意图到已配对集的最近邻打分 s(I) 可分（AUC 0.77）。s 低=远离既有结构、V 段负担重 | 新意图能否被先例覆盖的前置信号 |

## 二、现有流程的问题（实跑实证）↔ 论文能否解

### 问题 1：draft 慢（查手册多轮，单 yzg ~40min draft）
- **实证**：3 个慢 case 共同点=需求最简（1步空 expected）+场景最杂（递归/跨设备/tcp），draft 要现查手册推断"配什么+哪些 show 验证+dig 参数+断言哪字段"。fork 内部反复 grep。
- **论文能解（直接指导）**：✅ **推论 3.11 + 可达性 3.22 正中要害**。draft 慢的根因是**把 G/E 段也交给 LLM 现查现生成**——而论文证明 G 段（命令骨架/show 命令）可由**先例检索的经验众数**确定性给出（§3.14 骨架机制 ĝ_L）、E 段（IP）查表即得（引理3.9），不该让 LLM 逐轮 grep 手册推断。
- **修法指导**：draft 流程改为"**先 qa_lookup_pattern 拿先例骨架（G 段众数）+ 查拓扑表（E 段）→ 只在没先例覆盖时才查手册→ V 段交 LLM**"。论文 §4.5 实测：depth-2 前缀已把骨架从 41 种收窄到少数几种、贝叶斯可达正确率 0.44——即**先例命中时 G 段基本不用查手册**。慢的是没先例覆盖的 case（ρ_k 低），论文 §3.14 注明这是"先例库覆盖度不足"，解法是**扩库**而非让 LLM 硬查。

### 问题 2：grade "最弱拖垮全局"偏严（655248/681556：有强断言仍因配置存在性检查被 CUT）
- **实证**：grade 把"配置存在性检查(0.3)+强动态断言(0.9)"按最弱项判 CUT，即使强断言已覆盖核心行为。
- **论文能解（直接指导）**：✅ **三层分解给了 grade 判据的正确分层**。论文区分：G 段断言（验骨架/配置存在）本就是**零自由度的确定性部分**，E 段断言（验 IP）查表可定，**只有 V 段断言（验业务语义/解析结果）才是"覆盖目标行为"的关键**。grade 现在的错误是**把 G/E 段的配置存在性检查和 V 段的行为验证混在一个"最弱拖垮"池里打分**。
- **修法指导**：grade 按三层分别要求——G 段断言只要合法即可（不参与"覆盖度"评分）、E 段断言查表可定、**"覆盖目标行为"只由 V 段断言决定**。一条 case 只要 **V 段断言真覆盖了语义残差**（dig 验解析 IP、命中计数等）就该 PASS，附带的 G/E 配置存在性检查是健全性前置、不该拖垮全局。这正是论文推论3.11"不可约不确定性=H_V'"的直接应用——**质量门应只盯 V 段**。

### 问题 3：check_point 悬空崩框架（655203：check_point 紧跟无回显配置命令）
- **实证**：`check_point found "sdns on"` 跟在 `cmds_config`（写操作，无回显）后 → result=None → 框架 TypeError → 中断整包。
- **论文部分相关（概念支撑，非直接解）**：⚠️ 论文的 G 段理论解释了**为什么这是错的**——"sdns on" 是 G 段（配置动作），验证它"配成功"应通过 **E/V 段的可观测产物**（show 回显、dig 结果），而非对配置动作本身做 check_point。论文 §3.2 算子代数区分"配置算子"与"观测算子"。但**具体的"check_point 必须紧跟 show/dig"是框架工程约束，论文不涉及**。
- **修法**：emit 加结构校验门（工程修，论文只提供"为何配置动作不可直接断言"的概念依据）。

### 问题 4：dig 解析 fail 归因不清（grade PASS 但框架 fail 在 dig 没解析到 IP）
- **实证**：前3个 grade PASS 的 case 框架实跑 fail，fail 在 dig 没返回 172.16.35.231（真实后端 IP）。
- **论文能解（归因框架）**：✅ **§5.4 的错误归因分布（G错/E错/V错/环境瞬态四分）正是这个**。dig 没解析到正确后端 IP——按论文分类：若 IP 本身配错=E 错，若 sdns 配置链路没生效=G 错（骨架不全），若环境/网络不通=环境瞬态。论文给了**把 fail 精确归到哪一层的分类法**，正好用来重构 ist_verify 现在"断言失败 vs 环境瞬态"的粗二分。
- **修法指导**：ist_verify 的 fail 分类从二分→四分（G/E/V/瞬态），按论文 §5.4 归因。这也回答了"grade PASS 为何上机 fail"——grade 验的是 V 段断言质量（PASS 对），fail 在 G/E 段或环境（正交，§4.7+§3.10 证明两者独立）。

### 问题 5：合并 xlsx 一个 case 崩中断整包
- **论文管不着**：❌ 纯框架行为（pytest 一个 case 异常中断后续），与信息论分解无关。工程修（case 隔离/剔除崩的）。

### 问题 6：环境瞬态 fail（SSH 中断/dig 超时/DNS 失败）
- **论文部分相关**：⚠️ 论文 §5.4 把"环境瞬态"明确列为**不计入方法对比的第四类**，§5.7 也说环境失败是有效性威胁。即论文承认这类 fail 与编译质量无关、应隔离——支持我们"环境瞬态不回流重编译"的现有设计。但**具体怎么识别/重试是工程**。

## 三、对照结论

### 论文能直接指导解决的（理论有定量依据）
1. **draft 慢**（问题1）→ 推论3.11 + 可达性3.22：G/E 段该确定性产（检索先例骨架+查表），不该 LLM 现查；慢是先例库覆盖不足(ρ_k 低)，解法是扩库。
2. **grade 偏严**（问题2）→ 三层分解：质量门应只盯 V 段，G/E 段配置存在性检查不该拖垮全局。
3. **fail 归因粗**（问题4）→ §5.4 四层归因：ist_verify 的 fail 分类按 G/E/V/瞬态细化。

### 论文提供概念支撑但需工程实现的
4. check_point 悬空（问题3）→ G 段配置动作不可直接断言（概念），但校验门是工程。
5. 环境瞬态隔离（问题6）→ 论文承认应隔离（支持现设计），识别/重试是工程。

### 论文管不着的（纯工程/框架）
6. 合并 xlsx 崩整包（问题5）→ pytest 框架行为。

## 四、最高杠杆的应用（如果只做一件）

**用三层分解重构 draft 的生成顺序 + grade 的判据**（问题1+2，论文指导最直接）：

- **draft**：显式按 G→E→V 三段组织。G 段优先 `qa_lookup_pattern` 取先例骨架众数（命中即定，不查手册）、E 段查拓扑表、**只有 V 段交 LLM 现场填**。这把论文推论3.11 的"G/E 确定性、V 才自由"从分析框架变成生成器组织原则——直接治 draft 慢（少查手册多轮）+ 不稳定（G/E 不再随机）。
- **grade**：判据按三层分离，**"覆盖目标行为"只由 V 段断言判定**，G/E 段配置存在性检查降为健全性前置、不拖垮全局——治问题2 的偏严。

这两件正好是论文 §5.2 **Arm-L（分层生成）的核心机制**——论文已从理论（推论3.11）证明分层应降低不稳定、不损正确性。我们要做的是把"分层"从现在的"工具可用但 draft 没强制按 G/E/V 顺序组织"，落成 draft prompt 里**显式的三段生成纪律**。

## 五、不做的（避免过度应用）
- 不引入信息论度量（条件熵/碰撞熵）到运行时——那是论文的**分析**工具，不是生成器需要实时算的。运行时只需"G 段先检索、V 段才 LLM"这个**定性原则**，不需算 H_G/H_V。
- 不做 §5 双臂对照实验（你说那是另一任务）。
- 绝对值数字（ρ=0.618 等）是论文实证，不进我们的生成逻辑。
